📋 คู่มือตั้งค่าระบบด้วยตนเอง (Manual Setup)

คำแนะนำทีละขั้นตอนสำหรับตั้งค่าระบบ ปกติ ด้วยตัวแปรและการรันคำสั่งโดยตรง
⚙️ คู่มือสำหรับการติดตั้งแบบปกติ: หน้านี้จัดทำขึ้นเพื่อให้คุณตั้งค่าระบบบอตและแดชบอร์ดด้วยขั้นตอนมาตรฐานสำหรับนักพัฒนาทั่วไป โดยไม่มีการใช้สคริปต์อัตโนมัติ CMD ใดๆ เพื่อความยืดหยุ่นในการปรับแต่งส่วนหลังบ้านทั้งหมดครับ
🧭 สารบัญขั้นตอนการติดตั้ง:
  • 🔑 สเต็ปที่ 1: การขอข้อมูลและคีย์ API สำคัญ
  • 🗄️ สเต็ปที่ 2: การสร้างตารางฐานข้อมูลใน Supabase
  • 📝 สเต็ปที่ 3: การเขียนไฟล์ค่ากำหนดสภาพแวดล้อม (.env)
  • 🚀 สเต็ปที่ 4: การ Deploy ขึ้นระบบจริงบน Vercel
  • 🔌 สเต็ปที่ 5: การเชื่อมระบบ Webhooks
  • 🏁 สเต็ปที่ 6: การเริ่มดึงข้อมูลครั้งแรก (First Sync)
🔑 สเต็ปที่ 1: การขอข้อมูลและคีย์ API สำคัญ (ทีละขั้นตอนอย่างละเอียด)

ให้เปิดล็อกอิน (Login) บัญชีผู้ใช้งานค้างไว้ในเว็บเบราว์เซอร์ และคัดลอกค่ารหัสต่างๆ ด้านล่างนี้ไปจดลงในโปรแกรม Notepad บนเครื่องคอมพิวเตอร์ของคุณเตรียมไว้ครับ:

1.1 ลิงก์เชื่อมฐานข้อมูล Supabase (DATABASE_URL)

จุดประสงค์: ใช้เพื่อให้บอตสามารถเชื่อมต่อและเก็บบันทึกประวัติตั๋วงานทั้งหมดได้

  1. เข้าเว็บไซต์ Supabase.com -> คลิกปุ่มสีเขียว "Start your project" หรือสมัครสมาชิกผ่านบัญชี GitHub ให้เรียบร้อย
  2. เมื่อเข้ามาหน้าแรก ให้คลิกปุ่มสีเขียว "New Project" (สร้างโครงการใหม่) -> เลือกชื่อองค์กรของคุณ
  3. กรอกรายละเอียดโครงการใหม่:
    • Name (ชื่อโปรเจกต์): พิมพ์ตั้งชื่อตามชอบ เช่น Tasky-Jira-Bot
    • Database Password: ⚠️ สำคัญมาก: ตั้งรหัสผ่านฐานข้อมูลที่มีความปลอดภัย แล้วก๊อปปี้รหัสผ่านจริงตัวนี้จดแยกใส่ Notepad เก็บไว้ทันที (เพราะระบบจะไม่แสดงรหัสนี้ให้เห็นอีก และต้องนำไปใช้แทนที่ในภายหลัง)
    • Region: เลือกเป็น Singapore (ap-southeast-1) (เพื่อให้เซิร์ฟเวอร์อยู่ใกล้ไทยและทำงานได้ไวที่สุด)
    • Pricing Plan: เลือกเป็น Free
  4. คลิกปุ่มสีเขียว "Create new project" -> รอระบบเตรียมการประมาณ 1-2 นาที (สังเกตไอคอนวงกลมหมุนๆ จนกระทั่งสร้างโครงการเสร็จสมบูรณ์)
  5. เมื่อเข้ามาที่หน้าแรกของโครงการแล้ว มองไปที่แถบสีดำด้านบนสุดขวามือ คลิกปุ่มสีเขียวคำว่า "Connect" (อยู่ข้างชื่อโครงการของคุณ)
  6. ในป๊อปอัปสี่เหลี่ยมที่เด้งขึ้นมา ให้คลิกเลือกแท็บคำว่า "Direct" ที่อยู่ด้านบนสุด
  7. ในหน้าจอของแท็บ Direct ให้ระบุตั้งค่า 2 ข้อดังนี้:
    • Connection Method: ติ๊กเลือกที่หัวข้อ "Transaction pooler"
    • Type: คลิกเลือกที่คำว่า "URI"
  8. สังเกตกล่องสีดำที่มีลิงก์ขึ้นต้นด้วย postgres://... ให้คลิกปุ่ม "Copy" ที่อยู่มุมขวาของกล่องนั้นเพื่อคัดลอกลิงก์
  9. เอาลิงก์มาวางใน Notepad แล้วสังเกตตัวหนังสือคำว่า [YOUR-PASSWORD] ให้ลบคำนี้ออก (รวมทั้งวงเล็บเหลี่ยมด้วย) แล้วนำ รหัสผ่านจริง ที่คุณตั้งไว้ในขั้นตอนแรกสุดมาพิมพ์ใส่แทนที่ตรงจุดนั้นเป๊ะๆ
1.2 ข้อมูลการล็อกอินและเชื่อมต่อกับ Jira Cloud
  1. JIRA_DOMAIN (โดเมนของบริษัท): เปิดหน้าเบราว์เซอร์ใช้งาน Jira บอร์ดหลักปกติ สังเกตแถบที่อยู่เว็บด้านบน (URL) ก๊อปปี้เอาเฉพาะท่อนโดเมน เช่น mycompany.atlassian.net (ห้ามพิมพ์ https:// หรือเครื่องหมายขีดทับ / ต่อท้ายเด็ดขาด)
  2. JIRA_EMAIL: กรอกที่อยู่อีเมลที่คุณใช้เข้าใช้งานระบบ Jira ของที่ทำงานปกติ
  3. JIRA_API_TOKEN (รหัสผ่านเชื่อมโยงข้อมูล):
    • เข้าหน้าจัดการความปลอดภัยของ Atlassian: Atlassian API Tokens
    • คลิกปุ่มสีน้ำเงิน "Create API token"
    • ตั้งชื่อป้ายกำกับว่า Jira-GoogleChat-Bot แล้วกด Create
    • หน้าต่างจะแสดงรหัสคีย์ที่ขึ้นต้นด้วย ATATT... ให้กดปุ่ม "Copy" เก็บเข้า Notepad ทันที (⚠️ คำเตือน: รหัสนี้จะแสดงครั้งเดียวตอนสร้างเสร็จเท่านั้น หากปิดไปแล้วต้องสร้างใหม่)
  4. JIRA_PROJECT_KEY (คีย์ย่อโครงการ): สังเกตชื่อรหัสตั๋วงานใน Jira ของคุณ เช่น ตั๋วงานมีชื่อรหัสว่า DEV-120 หรือ TES-99 ให้เอาเฉพาะส่วนหน้าขีดคือ DEV หรือ TES (พิมพ์ตัวพิมพ์ใหญ่ทั้งหมด)
1.3 ลิงก์ห้องแชท Google Chat Webhook URL

จุดประสงค์: เป็นท่อปลายทางเพื่อให้บอตสามารถส่งข้อความไปหาคนในทีมได้

  1. เปิดหน้าแอป Google Chat ในคอมพิวเตอร์ของคุณ
  2. เอาเมาส์ไปชี้ที่ชื่อห้องกลุ่ม (Space) ของทีมคุณ -> คลิกที่เครื่องหมายสามจุด ... (หรือคลิกที่ชื่อกลุ่มด้านบนสุดของหน้าจอแชท) -> เลือกเมนู "Apps & integrations" (แอปและการบูรณาการ)
  3. คลิกเลือกเมนูย่อย "Webhooks"
  4. คลิกปุ่ม "Add webhook" (หรือปุ่มเพิ่ม) -> ตั้งชื่อโปรไฟล์ของบอต เช่น Tasky Alert Bot -> คลิกปุ่ม Save (บันทึก)
  5. กดปุ่ม "Copy" ลิงก์ที่เด้งขึ้นมาเก็บไว้ (ขึ้นต้นด้วย https://chat.googleapis.com/v1/spaces/...)
1.4 รหัส Gemini API Key (คีย์สมองบอต AI)

จุดประสงค์: เพื่อใช้ปัญญาประดิษฐ์มาสรุปแจกแจงงานด่วนและวิเคราะห์ทีมงาน

  1. เปิดหน้าเว็บ Google AI Studio แล้วล็อกอินด้วยบัญชี Gmail/Google ทั่วไป
  2. มองหาแท็บเมนูด้านซ้ายบนสุด คลิกที่เมนูรูปไอคอนกุญแจเขียนว่า "Get API key"
  3. คลิกปุ่มสีน้ำเงินคำว่า "Create API key"
  4. หน้าต่างย่อยจะเด้งขึ้นมา ให้เลือกสร้างคีย์ในโปรเจกต์ใหม่ หรือเลือกโปรเจกต์เดิมที่มีอยู่ แล้วคลิกยืนยันสร้าง
  5. คลิกปุ่ม "Copy" รหัสคีย์ยาวๆ ที่ระบบสรุปให้มา (รหัสจะขึ้นต้นด้วยตัวอักษรย่อ AQ. หรือ AL. เสมอ) ก๊อปปี้ไปวางรอไว้ใน Notepad ครับ
🗄️ สเต็ปที่ 2: การสร้างตารางฐานข้อมูลใน Supabase (Database Setup)

เนื่องจากเป็นขั้นตอนปกติ ให้คุณคัดลอกสคริปต์ SQL ด้านล่างนี้ไปสั่งรันตารางในหน้าเว็บโดยตรงครับ:

  1. เปิดหน้าโครงการในเว็บ Supabase -> คลิกเลือกเมนู "SQL Editor" ที่แถบเมนูด้านซ้าย
  2. คลิกสร้าง "New Query" หน้าเอกสารเปล่าใหม่
  3. คัดลอกสคริปต์ด้านล่างนี้ไปแปะในหน้าต่างพิมพ์คำสั่ง:
CREATE TABLE IF NOT EXISTS team_members (
  id SERIAL PRIMARY KEY,
  nickname VARCHAR(100) UNIQUE NOT NULL,
  jira_display_name VARCHAR(255) UNIQUE NOT NULL,
  email VARCHAR(255),
  webhook_url TEXT
);

CREATE TABLE IF NOT EXISTS activity_logs (
  id SERIAL PRIMARY KEY,
  timestamp TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
  username VARCHAR(100) NOT NULL,
  role VARCHAR(100) NOT NULL,
  action VARCHAR(100) NOT NULL,
  ticket_key VARCHAR(100),
  details TEXT
);

CREATE TABLE IF NOT EXISTS ticket_status_cache (
  ticket_key VARCHAR(100) PRIMARY KEY,
  status VARCHAR(100) NOT NULL,
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS user_spaces_map (
  nickname VARCHAR(100) PRIMARY KEY,
  space_name TEXT NOT NULL
);

CREATE TABLE IF NOT EXISTS users (
  id SERIAL PRIMARY KEY,
  username VARCHAR(100) UNIQUE NOT NULL,
  password_hash TEXT NOT NULL,
  salt VARCHAR(200) NOT NULL,
  name VARCHAR(255) NOT NULL,
  email VARCHAR(255),
  role VARCHAR(100) NOT NULL DEFAULT 'Admin',
  is_approved BOOLEAN NOT NULL DEFAULT TRUE,
  jira_display_name VARCHAR(255),
  jira_account_id VARCHAR(255),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS tickets (
  key VARCHAR(100) PRIMARY KEY,
  summary VARCHAR(255) NOT NULL,
  status VARCHAR(100) NOT NULL,
  issuetype VARCHAR(100) NOT NULL,
  priority VARCHAR(100) NOT NULL,
  assignee VARCHAR(255),
  reporter VARCHAR(255),
  created TIMESTAMP WITH TIME ZONE,
  duedate DATE,
  resolved TIMESTAMP WITH TIME ZONE,
  parent VARCHAR(100),
  description TEXT,
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
  1. คลิกปุ่มสีเขียวคำว่า "Run" (หรือกด Ctrl + Enter บนแป้นพิมพ์) เพื่อสร้างตารางทั้งหมดเสร็จสิ้น
  2. การสร้างบัญชีผู้ดูแลระบบ (Admin User):
    • ลบคำสั่งสร้างตารางเดิมในช่อง SQL Editor ออกให้หมด
    • คัดลอกคำสั่งรันด้านล่างนี้ลงไปแปะในหน้าต่างเดิม: 
      -- คำสั่งเพิ่มผู้ใช้ Admin เข้าสู่ตาราง เพื่อนำไปใช้ล็อกอินหน้า Dashboard
-- (โดยบัญชีนี้จะมีชื่อล็อกอินว่า: admin และรหัสผ่านคือ: admin123)
INSERT INTO users (username, password_hash, salt, name, email, role, is_approved)
VALUES (

  '457bd77b738e4a9e52541db8b5df40d7c3dbb9fa54a19b8ee49195d5279b908b9826f4974917540788102a90da3d1796c9c614b8a4f9104b2b467651048b26ef', 
  '73238596645391a3c617c72957bdc026', 
  'Administrator',

    • คลิกปุ่มสีเขียว "Run" อีกครั้งเพื่อเปิดบัญชี
    • 💡 แนะนำเพิ่มเติม: บัญชีนี้ใช้สำหรับล็อกอินหน้า Dashboard โดยใช้ username: admin และ password: admin123 เมื่อติดตั้งหน้าเว็บจนใช้งานได้แล้ว ค่อยกดเข้าสู่ระบบไปเปลี่ยนรหัสผ่านใหม่ได้ทันทีครับ
📝 สเต็ปที่ 3: การเขียนไฟล์ค่ากำหนดสภาพแวดล้อม (.env)

เมื่อจัดแจงข้อมูลส่วนหลังบ้านเสร็จแล้ว ต่อมาเราจะนำค่าทั้งหมดมาผูกลงในไฟล์ตั้งค่าของโปรเจกต์ครับ:

  1. เปิดโฟลเดอร์หลักของโปรเจกต์คุณขึ้นมาในเครื่องคอมพิวเตอร์
  2. เปิดการแสดงผลนามสกุลไฟล์บนระบบ Windows (สำคัญมาก):
    • หากใช้ Windows 11: คลิกที่เมนู View ด้านบนสุด -> ชี้ไปที่ Show -> ติ๊กเลือก "File name extensions" (นามสกุลไฟล์)
    • หากใช้ Windows 10: คลิกที่แท็บ View ด้านบนสุดของหน้าต่าง -> ติ๊กถูกที่ช่อง "File name extensions" (นามสกุลไฟล์)
    • ⚠️ หากไม่ทำข้อนี้: เวลาเปลี่ยนชื่อไฟล์อาจจะกลายเป็น .env.txt ซึ่งระบบจะไม่ยอมรับและรันงานไม่ผ่านครับ
  3. ค้นหาไฟล์ตัวอย่างที่ชื่อ .env.example -> คลิกขวาเลือก Copy (คัดลอก) และกด Paste (วาง) ลงในโฟลเดอร์เดิม
  4. เปลี่ยนชื่อไฟล์ที่เพิ่งคัดลอกใหม่นี้ให้เหลือแค่ `.env` (ลบข้อความ .example หรือ .txt ออกให้หมด เหลือเพียงจุดทศนิยมและคำว่า env)
  5. คลิกขวาที่ไฟล์ .env เลือก Open with -> โปรแกรม Notepad (หรือ VS Code) แล้วแก้ไขข้อมูลใส่คีย์ที่จดไว้ลงไปดังนี้ครับ:
DATABASE_URL="ลิงก์เชื่อม Supabase จากข้อ 1.1"

JIRA_PROJECT_KEY="รหัสตัวย่อโครงการ เช่น DEV"

GOOGLE_CHAT_WEBHOOK_URL="ลิงก์ส่งข้อความจากข้อ 1.3"

LLM_PROVIDER="gemini"
GEMINI_API_KEY="คีย์ AI จากข้อ 1.4"
GEMINI_MODEL="gemini-3.5-flash"

BOT_NAME="TaskyBot"
🚀 สเต็ปที่ 4: การ Deploy ขึ้นระบบจริงบน Vercel
  1. นำซอร์สโค้ดโปรเจกต์ทั้งหมดอัปโหลดขึ้น GitHub Repository ส่วนตัว (Private Repository) ของคุณ
  2. ไปที่เว็บ **Vercel.com** ล็อกอินด้วย GitHub -> คลิกปุ่ม **"Add New"** -> **"Project"**
  3. กดปุ่ม "Import" โปรเจกต์ดังกล่าวจากรายชื่อของคุณ
  4. เลื่อนหาเมนูย่อย **Environment Variables** นำค่าคีย์ทั้งหมดที่เขียนไว้ในไฟล์ .env ในข้อ 3 มากรอกใส่ให้ครบถ้วน
  5. คลิกปุ่มสีน้ำเงินคำว่า "Deploy" และก๊อปปี้ลิงก์ที่เว็บที่ได้ (เช่น https://my-jira-bot.vercel.app) เก็บไว้
🔌 สเต็ปที่ 5: การเชื่อมระบบ Webhooks
5.1 ตั้งค่าเว็บฮุคฝั่ง Jira Cloud:

เปิดลิงก์ตั้งค่านี้ในเบราว์เซอร์: https://[โดเมน_Jira_ของคุณ]/plugins/servlet/webhooks

  1. คลิกปุ่ม "Create a Webhook" ขวาบน
  2. ตั้งค่ารายละเอียดดังนี้:
    • Name: Sync to Dashboard
    • URL: ลิงก์ Vercel ของคุณต่อท้ายด้วย /api/jira/webhook (เช่น https://my-jira-bot.vercel.app/api/jira/webhook)
    • JQL Filter: ระบุคำสั่งสูตรกรอง project = "รหัสย่อโปรเจกต์ของคุณ" (เช่น project = "DEV")
    • Events: เลื่อนลงหาหมวด Issue ติ๊กถูกที่ช่อง **Created**, **Updated**, และ **Deleted**
  3. คลิกปุ่มสีน้ำเงินด้านล่างสุด "Create"
5.2 ตั้งค่าเว็บฮุคฝั่ง Google Chat API:
  1. เข้าเว็บ Google Cloud Console เลือกโปรเจกต์งานของคุณ
  2. ค้นหาเปิดบริการ "Google Chat API" แล้วกดเปิดใช้งาน (Enable)
  3. ไปที่เมนู "Configuration" ด้านซ้าย กรอกโปรไฟล์บอตตามชอบ
  4. ที่หัวข้อ **Connection settings** เลือก **"Webhook URL"**
  5. วางที่อยู่อีเมลลิงก์เว็บ Vercel ของคุณ และใส่ต่อท้ายด้วย /api/webhook (เช่น https://my-jira-bot.vercel.app/api/webhook)
  6. คลิกปุ่ม "Save" ล่างสุด
