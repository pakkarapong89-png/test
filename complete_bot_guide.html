                                                                       │

🛠️ ส่วนที่ 1: การสร้างและ Deploy Webhook บน Google Apps Script

ในส่วนนี้เราจะสร้างตัวรับสัญญาณจาก Google Chat เพื่อส่งต่อมาที่เว็บ Next.js ของเรา

1.1 สร้างโปรเจกต์ Apps Script

1.เข้าไปที่เว็บ 👉 Google Apps Script (ล็อกอินด้วยบัญชี Google ขององค์กรหรือบัญชีที่คุณใช้ใน Google Chat)
2.คลิกปุ่ม New project (โครงการใหม่) ด้านบนซ้าย
3.เปลี่ยนชื่อโครงการจาก "Untitled project" เป็น taskyapp Bot Bridge (คลิกที่ชื่อเพื่อเปลี่ยน)

1.2 วางโค้ดเชื่อมต่อ Webhook

1.ในช่องเขียนโค้ด ให้ลบฟังก์ชัน myFunction() เดิมออกทั้งหมด
2.คัดลอกโค้ดด้านล่างนี้ไปวางแทนที่:
function onMessage(event) {
  // ⚠️ สำคัญมาก: แก้ไข URL ด้านล่างเป็น URL เว็บ Vercel ของคุณเอง (ต้องลงท้ายด้วย /api/webhook)
  var url = "https://[ลิงก์เว็บของคุณ]/api/webhook";
  
  var payload = {
    message: {
      text: event.message.text,
      sender: {
        displayName: event.user.displayName
      },
      space: {
        name: event.space.name
      }
    }
  };
  
  var options = {
    method: "post",
    contentType: "application/json",
    payload: JSON.stringify(payload),
    muteHttpExceptions: true
  };
  
  try {
    var response = UrlFetchApp.fetch(url, options);
    var result = JSON.parse(response.getContentText());
    
    return {
      text: result.text || "⚠️ บอทได้รับคำสั่งแล้วแต่ไม่สามารถประมวลผลคำตอบได้"
    };
  } catch (error) {
    return {
      text: "⚠️ เกิดข้อผิดพลาดในการเชื่อมต่อเซิร์ฟเวอร์ Next.js: " + error.message
    };
  }
}
3.แก้ไขโค้ดบรรทัดที่ 3: เปลี่ยน "https://[ลิงก์เว็บของคุณ]/api/webhook" เป็นลิงก์เว็บจริงของคุณที่ Deploy บน Vercel เช่น:

var url = "https://taskyapp-demo.vercel.app/api/webhook";

4.กดปุ่ม 💾 Save Project (รูปแผ่นดิสก์ด้านบน)

1.3 Deploy เพื่อรับ ID

1.คลิกปุ่ม Deploy (การใช้งานจริง) ที่มุมขวาบน -> เลือก New deployment (การใช้งานจริงรายการใหม่)
2.คลิกไอคอนฟันเฟือง ⚙️ ข้างข้อความ "Select type" -> เลือก Add-on (ส่วนเสริม) -> จะได้หน้าต่างที่มีประเภทย่อยเป็น Google Chat App
3.ใส่รายละเอียดช่อง Description ว่า Bridge for taskyapp
4.กดปุ่ม Deploy
5.ระบบจะแสดงหน้าต่างยืนยันความสำเร็จ ให้คัดลอกค่า Deployment ID (จะมีลักษณะเป็นรหัสยาวๆ) เอาไว้ใน Notepad

🌐 ส่วนที่ 2: การเปิดใช้และตั้งค่า Google Chat API บน Google Cloud Console

ส่วนนี้เป็นการลงทะเบียนเพื่อให้ระบบของ Google Chat รู้จักกับบอทของเราและลิงก์เข้ากับ Apps Script

2.1 เข้าสู่ Google Cloud Console

1.เข้าไปที่ 👉 Google Cloud Console
2.หากยังไม่มี Project ให้กดสร้าง Project ใหม่ โดยตั้งชื่อว่า taskyapp-project

2.2 เปิดใช้งาน Google Chat API

1.มองหาช่องค้นหาด้านบนสุดของหน้า Cloud Console
2.พิมพ์คำว่า Google Chat API แล้วกดค้นหา
3.คลิกเข้าหน้า Google Chat API แล้วกดปุ่ม Enable (เปิดใช้งาน)

2.3 ตั้งค่า Configuration ของบอท

เมื่อเปิดใช้งานแล้ว ให้คลิกแท็บ Configuration (การตั้งค่า) ทางด้านซ้าย และกรอกข้อมูลตามตารางนี้:

ช่องป้อนข้อมูลค่าที่ต้องกรอกคำอธิบาย
App nametaskyappชื่อที่จะปรากฏในห้องแชท Google Chat
Avatar URL(เว้นว่างไว้ หรือใส่ URL รูปภาพโลโก้ของคุณ)รูปโปรไฟล์ของบอท
Descriptionบอทอัจฉริยะสำหรับจัดการงาน Jiraคำอธิบายการทำงานของบอท
Functionalityติ๊กเลือก [x] Receive 1:1 messages และ [x] Join spaces and group conversationsอนุญาตให้คุยแบบตัวต่อตัวและดึงเข้าห้องกลุ่มได้
Connection settingsเลือก Apps Script projectเชื่อมโยงบอทเข้ากับสคริปต์ที่เราเขียน
Deployment IDวาง Deployment ID ที่คัดลอกมาจากส่วนที่ 1.3รหัสประจำตัวของ Apps Script
Visibilityเลือก Everyone in your organization (ทุกคนในองค์กรของคุณ) หรือเลือกจำกัดเฉพาะคุณสิทธิ์การเข้าถึงตัวบอท
1.เลื่อนลงล่างสุดแล้วกดปุ่ม Save (บันทึก)

2.4 เชิญบอทเข้าห้องแชท

1.เปิดแอป Google Chat
2.เลือกห้องกลุ่ม (Space) หรือเปิดห้องคุยแบบ 1:1 ที่ต้องการใช้งานบอท
3.หากอยู่ในห้องกลุ่ม ให้กดปุ่มหัวข้อห้อง -> เลือก Apps & integrations (แอปและการผสานรวม)
4.กด Add apps (เพิ่มแอป) -> พิมพ์ค้นหาบอทชื่อ taskyapp
5.กด Add เพื่อเพิ่มบอทเข้ามาในห้อง

🔗 ส่วนที่ 3: การเชื่อมต่อ Jira Webhook กลับมายังห้องแชท

ขั้นตอนนี้จะทำเพื่อให้เมื่อเราทำการแก้ไขข้อมูลบนบอร์ด Jira ตัวบอร์ดจะยิงสัญญาณกลับมาแจ้งเตือนที่ห้อง Google Chat ทันที
1.ไปที่หน้าบอร์ด Jira Cloud ของคุณ
2.คลิกปุ่มฟันเฟือง ⚙️ Settings ที่มุมขวาบน -> เลือกเมนู System
3.เลื่อนแถบเมนูด้านซ้ายลงไปข้างล่างสุด หาหัวข้อ Webhooks (อยู่ใต้เมนู Advanced)
4.คลิกปุ่ม Create a Webhook (สร้าง Webhook) ด้านขวาบน
5.กรอกข้อมูลการตั้งค่า:

- Name: taskyapp Notifier

- Status: เลือก Enabled

- URL: กรอกลิงก์เว็บ Vercel ของคุณตามด้วย /api/jira/webhook เช่น

https://taskyapp-demo.vercel.app/api/jira/webhook

6.เลื่อนลงมาที่หัวข้อ Events มองหาหมวด Issue และติ๊กเลือกเหตุการณ์ดังนี้:

- [x] created (ทำงานเมื่อมีการสร้างตั๋วงานใหม่)

- [x] updated (ทำงานเมื่อมีการเปลี่ยนสถานะ ย้ายบอร์ด หรือเปลี่ยนชื่อผู้รับผิดชอบ)

7.เลื่อนลงไปด้านล่างสุด แล้วกดปุ่ม Create (สร้าง)

💬 ส่วนที่ 4: คู่มือการพิมพ์คำสั่งสั่งงานบอท (ภาษาไทย & NLP)

บอทตัวนี้มีสมองกล AI (Google Gemini) ในตัว คุณสามารถพิมพ์คุยภาษาไทยอย่างเป็นธรรมชาติได้โดยไม่ต้องพิมพ์คำสั่งตายตัว แต่ต้องขึ้นต้นด้วย @taskyapp ทุกครั้ง

4.1 การตั้งค่าชื่อเล่นของทีม (ก่อนเริ่มใช้คำสั่งมอบหมายงาน)

เพื่อให้บอทรู้ว่า "สมชาย" คือบัญชีผู้ใช้ใดใน Jira ก่อนสั่งงานให้ผู้ใช้ล็อกอินหน้า Dashboard ของเว็บ ไปที่แท็บ Team และแอดข้อมูล ชื่อเล่น (Nickname) ลิงก์เข้ากับ ชื่อจริงบน Jira เสียก่อน

4.2 คำสั่งพื้นฐานที่บอทเข้าใจทันที:

1️⃣ คำสั่งสร้างงานใหม่ (Create Issue)

•รูปแบบทั่วไป: พิมพ์ @taskyapp สร้างงาน [ชื่อหัวข้อ] [รายละเอียดเพิ่มเติม]
•ตัวอย่างประโยคธรรมชาติ:

2️⃣ คำสั่งสร้างหลายงานพร้อมกัน (Bulk Create)

ใช้คำว่า "และ" หรือ "and" เพื่อแบ่งตั๋วงาน

•ตัวอย่างประโยคธรรมชาติ:

🔧 ส่วนที่ 5: การแก้ปัญหาเกี่ยวกับบอท (Troubleshooting)

❌ ปัญหา: พิมพ์หาบอทแล้วบอทเงียบ ไม่มีข้อความตอบกลับใดๆ

1.ตรวจสอบความถูกต้องใน Google Cloud Platform:

- มั่นใจว่าคุณใส่ Deployment ID ในหน้า Configuration ถูกต้องและไม่มีเว้นวรรค

- ตรวจสอบว่าใน Apps Script ได้กดบันทึกและกด Deploy เรียบร้อยแล้ว

2.ตรวจสอบปัญหาเว็บ Vercel ค้าง (Cold Start / Serverless Timeout):

- บางครั้งการรันบอทครั้งแรกหลังจากเว็บไม่มีกิจกรรมมานาน อาจใช้เวลาประมวลผลนานเกิน 10 วินาทีจนเกิด Timeout

- วิธีแก้: ลองเปิดเว็บหน้า Dashboard ของคุณผ่านเบราว์เซอร์เพื่อให้ระบบตื่นตัว จากนั้นทดลองพิมพ์หาบอทใหม่อีกครั้ง

3.ตรวจสอบ Apps Script Executions:

- เปิดหน้า Apps Script -> คลิกสัญลักษณ์ 📊 Executions (การเรียกใช้) ด้านซ้าย

- ดูว่าการทำงานมีสถานะขึ้นว่า Failed (ล้มเหลว) หรือไม่ หากมีให้คลิกเข้าไปอ่าน Error Message

❌ ปัญหา: บอทตอบว่า "⚠️ เกิดข้อผิดพลาดในการเชื่อมต่อเซิร์ฟเวอร์ Next.js: ..."

•สาเหตุ: ตัว Apps Script ยิง HTTP ไปหาเซิร์ฟเวอร์เว็บไม่ได้ หรือเว็บปลายทางตอบกลับมาเป็น Error 500
•วิธีแก้ไข:

1. นำ URL ที่เขียนในบรรทัดที่ 3 ของ Apps Script มาเช็กความถูกต้อง ต้องเขียนในรูปแบบ https://[โดเมนของคุณ].vercel.app/api/webhook เท่านั้น

2. ลองทดสอบเปิดเบราว์เซอร์ไปที่ https://[โดเมนของคุณ].vercel.app/api/health เพื่อเช็กสถานะการเชื่อมต่อฐานข้อมูลและตัวแปรระบบว่าพร้อมใช้งานหรือไม่

❌ ปัญหา: บอทตอบกลับว่าสร้างงานสำเร็จ แต่ไม่มีงานปรากฏขึ้นในระบบ Jira

•สาเหตุ: การตั้งค่า JIRA_API_TOKEN หรือ JIRA_PROJECT_KEY ในตัวแปรระบบ (.env) บน Vercel ไม่ถูกต้อง
•วิธีแก้ไข:

1. ตรวจสอบให้แน่ใจว่าอีเมลและ Token ของ Jira ถูกต้อง

2. รหัสโครงการ (Project Key) ต้องเป็นตัวพิมพ์ใหญ่ตรงเป๊ะ เช่น KAN ไม่ใช่ kan หรือ Kan

❌ ปัญหา: มีข้อความแจ้งเตือนความเคลื่อนไหวแจ้งเข้ามาซ้ำ 2 รอบในห้องแชท

•คำอธิบาย: ระบบบอทเวอร์ชันก่อนหน้านี้แจ้งเตือนทุกครั้งที่มีการอัปเดตฟิลด์ใดๆ บน Jira (ซึ่งเวลาเราลากตั๋วเปลี่ยนสถานะ Jira จะแก้ไขตำแหน่งลำดับ หรือ Rank ทำให้แจ้งเตือนเด้งซ้ำ)
•สถานะ: ✅ ได้รับการแก้ไขเรียบร้อยแล้ว โดยบอทเวอร์ชันล่าสุดจะปิดการแจ้งเตือนเรื่องตำแหน่งลำดับภายในบอร์ด และจะส่งแจ้งเตือนเฉพาะกรณีที่สถานะ ผู้รับผิดชอบ หรือชื่อหัวข้องานเปลี่ยนไปจริงเท่านั้น
