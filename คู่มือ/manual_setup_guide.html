📋 คู่มือตั้งค่าระบบด้วยตนเอง (Manual Setup)

คำแนะนำทีละขั้นตอนสำหรับตั้งค่าระบบ ปกติ ด้วยตัวแปรและการรันคำสั่งโดยตรง
⚙️ คู่มือสำหรับการติดตั้งแบบปกติ: หน้านี้จัดทำขึ้นเพื่อให้คุณตั้งค่าระบบบอตและแดชบอร์ดด้วยขั้นตอนมาตรฐานสำหรับนักพัฒนาทั่วไป โดยไม่มีการใช้สคริปต์อัตโนมัติ CMD ใดๆ เพื่อความยืดหยุ่นในการปรับแต่งส่วนหลังบ้านทั้งหมดครับ
🧭 สารบัญขั้นตอนการติดตั้ง:
  • 🔑 สเต็ปที่ 1: การขอข้อมูลและคีย์ API สำคัญ
  • 🗄️ สเต็ปที่ 2: การสร้างตารางฐานข้อมูลใน Supabase
  • 📝 สเต็ปที่ 3: การเขียนไฟล์ค่ากำหนดสภาพแวดล้อม (.env)
  • 🚀 สเต็ปที่ 4: การ Deploy ขึ้นระบบจริงบน Vercel
  • 🔌 สเต็ปที่ 5: การเชื่อมระบบ Webhooks
  • 🏁 สเต็ปที่ 6: การเริ่มดึงข้อมูลครั้งแรก (First Sync)
🔑 สเต็ปที่ 1: การขอข้อมูลและคีย์ API สำคัญ

ล็อกอินเข้าบัญชีผู้ให้บริการของแต่ละที่ และคัดลอกรหัสสำคัญจดเก็บลงใน Notepad ล่วงหน้าดังนี้ครับ:

1.1 ลิงก์ฐานข้อมูล Supabase (DATABASE_URL)
  1. เปิดหน้าโครงการใน Supabase.com
  2. คลิกปุ่มสีเขียว "Connect" ด้านบนสุดของโปรเจกต์
  3. เลือกแท็บ "Direct" ในหน้าต่างย่อยด้านบนสุด
  4. ติ๊กหัวข้อ Connection Method ให้เป็น "Transaction pooler"
  5. เลือก Type เป็น "URI" และกดปุ่ม Copy ลิงก์ยาวที่ขึ้นต้นด้วย postgres://...
  6. นำค่า [YOUR-PASSWORD] ในข้อความลิงก์ออกแล้วพิมพ์แทนที่ด้วยรหัสผ่านจริงที่คุณตั้งค่าไว้
1.2 ข้อมูลฝั่ง Jira Cloud
  1. JIRA_DOMAIN: โดเมนหลักของทีม เช่น myteam.atlassian.net (ห้ามพิมพ์ https หรือทับ)
  2. JIRA_EMAIL: อีเมลที่ผูกเข้าใช้งาน Jira
  3. JIRA_API_TOKEN: สร้างได้จากหน้าเว็บ Atlassian API Tokens กดปุ่ม Create API token แล้ว Copy รหัสที่ขึ้นต้นด้วย ATATT...
  4. JIRA_PROJECT_KEY: รหัสย่อหน้าตั๋วงาน เช่น โครงการงานรหัส DEV-452 ค่ารหัสย่อคือ DEV
1.3 ลิงก์ Google Chat Webhook URL
  1. เปิดแอป Google Chat ในคอมพิวเตอร์ของคุณ
  2. เข้าไปใน Space ห้องแชทกลุ่ม -> คลิกชื่อห้องด้านบนสุด -> เลือก **Apps & integrations** -> เลือก **Webhooks**
  3. กดปุ่มเพิ่ม webhook และก๊อปปี้ลิงก์ที่ระบบสร้างให้เก็บไว้
1.4 รหัส Gemini API Key
  1. เปิดหน้าเว็บ Google AI Studio
  2. คลิก Get API key ด้านบนซ้าย -> คลิกปุ่มสีน้ำเงิน Create API key -> คัดลอกรหัสคีย์ที่ขึ้นต้นด้วย AQ. หรือ AL.
🗄️ สเต็ปที่ 2: การสร้างตารางฐานข้อมูลใน Supabase (Database Setup)

เนื่องจากเป็นขั้นตอนปกติ ให้คุณคัดลอกสคริปต์ SQL ด้านล่างนี้ไปสั่งรันตารางในหน้าเว็บโดยตรงครับ:

  1. เปิดหน้าโครงการในเว็บ Supabase -> คลิกเลือกเมนู "SQL Editor" ที่แถบเมนูด้านซ้าย
  2. คลิกสร้าง "New Query" หน้าเอกสารเปล่าใหม่
  3. คัดลอกสคริปต์ด้านล่างนี้ไปแปะในหน้าต่างพิมพ์คำสั่ง:
CREATE TABLE IF NOT EXISTS team_members (
  id SERIAL PRIMARY KEY,
  nickname VARCHAR(100) UNIQUE NOT NULL,
  jira_display_name VARCHAR(255) UNIQUE NOT NULL,
  email VARCHAR(255),
  webhook_url TEXT
);

CREATE TABLE IF NOT EXISTS activity_logs (
  id SERIAL PRIMARY KEY,
  timestamp TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
  username VARCHAR(100) NOT NULL,
  role VARCHAR(100) NOT NULL,
  action VARCHAR(100) NOT NULL,
  ticket_key VARCHAR(100),
  details TEXT
);

CREATE TABLE IF NOT EXISTS ticket_status_cache (
  ticket_key VARCHAR(100) PRIMARY KEY,
  status VARCHAR(100) NOT NULL,
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS user_spaces_map (
  nickname VARCHAR(100) PRIMARY KEY,
  space_name TEXT NOT NULL
);

CREATE TABLE IF NOT EXISTS users (
  id SERIAL PRIMARY KEY,
  username VARCHAR(100) UNIQUE NOT NULL,
  password_hash TEXT NOT NULL,
  salt VARCHAR(200) NOT NULL,
  name VARCHAR(255) NOT NULL,
  email VARCHAR(255),
  role VARCHAR(100) NOT NULL DEFAULT 'Admin',
  is_approved BOOLEAN NOT NULL DEFAULT TRUE,
  jira_display_name VARCHAR(255),
  jira_account_id VARCHAR(255),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS tickets (
  key VARCHAR(100) PRIMARY KEY,
  summary VARCHAR(255) NOT NULL,
  status VARCHAR(100) NOT NULL,
  issuetype VARCHAR(100) NOT NULL,
  priority VARCHAR(100) NOT NULL,
  assignee VARCHAR(255),
  reporter VARCHAR(255),
  created TIMESTAMP WITH TIME ZONE,
  duedate DATE,
  resolved TIMESTAMP WITH TIME ZONE,
  parent VARCHAR(100),
  description TEXT,
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
  1. คลิกปุ่มสีเขียวคำว่า "Run" เพื่อสร้างตารางทั้งหมดเสร็จสิ้น
  2. การเพิ่มบัญชี Admin บอร์ด: ไปที่เมนู **Table Editor** ด้านซ้าย -> เลือกตาราง **users** -> คลิกปุ่ม **Insert row** เพื่อเพิ่มชื่อและรหัสล็อกอินของผู้ดูแลหน้าเว็บแดชบอร์ดโดยตรงได้เลยครับ
📝 สเต็ปที่ 3: การเขียนไฟล์ค่ากำหนดสภาพแวดล้อม (.env)

สร้างไฟล์ชื่อ .env ไว้ที่ระดับโฟลเดอร์นอกสุดของโปรเจกต์คุณ และระบุค่าข้อมูลดังนี้:

DATABASE_URL="ลิงก์เชื่อม Supabase จากข้อ 1.1"

JIRA_DOMAIN="ชื่อโดเมนจากข้อ 1.2"
JIRA_EMAIL="อีเมลเข้าใช้งาน Jira"
JIRA_API_TOKEN="รหัสผ่าน API Token ขึ้นต้นด้วย ATATT..."
JIRA_PROJECT_KEY="รหัสตัวย่อโครงการ เช่น DEV"

GOOGLE_CHAT_WEBHOOK_URL="ลิงก์ส่งข้อความจากข้อ 1.3"

LLM_PROVIDER="gemini"
GEMINI_API_KEY="คีย์ AI จากข้อ 1.4"
GEMINI_MODEL="gemini-3.5-flash"

SESSION_SECRET="สุ่มข้อความยาว 32 ตัวเพื่อใช้ความปลอดภัยในเว็บ"
CRON_SECRET="สุ่มข้อความสั้น 16 ตัวเพื่อใช้เป็นความลับสั่งรัน Cron"
BOT_NAME="TaskyBot"
🚀 สเต็ปที่ 4: การ Deploy ขึ้นระบบจริงบน Vercel
  1. นำซอร์สโค้ดโปรเจกต์ทั้งหมดอัปโหลดขึ้น GitHub Repository ส่วนตัว (Private Repository) ของคุณ
  2. ไปที่เว็บ **Vercel.com** ล็อกอินด้วย GitHub -> คลิกปุ่ม **"Add New"** -> **"Project"**
  3. กดปุ่ม "Import" โปรเจกต์ดังกล่าวจากรายชื่อของคุณ
  4. เลื่อนหาเมนูย่อย **Environment Variables** นำค่าคีย์ทั้งหมดที่เขียนไว้ในไฟล์ .env ในข้อ 3 มากรอกใส่ให้ครบถ้วน
  5. คลิกปุ่มสีน้ำเงินคำว่า "Deploy" และก๊อปปี้ลิงก์ที่เว็บที่ได้ (เช่น https://my-jira-bot.vercel.app) เก็บไว้
🔌 สเต็ปที่ 5: การเชื่อมระบบ Webhooks
5.1 ตั้งค่าเว็บฮุคฝั่ง Jira Cloud:

เปิดลิงก์ตั้งค่านี้ในเบราว์เซอร์: https://[โดเมน_Jira_ของคุณ]/plugins/servlet/webhooks

  1. คลิกปุ่ม "Create a Webhook" ขวาบน
  2. ตั้งค่ารายละเอียดดังนี้:
    • Name: Sync to Dashboard
    • URL: ลิงก์ Vercel ของคุณต่อท้ายด้วย /api/jira/webhook (เช่น https://my-jira-bot.vercel.app/api/jira/webhook)
    • JQL Filter: ระบุคำสั่งสูตรกรอง project = "รหัสย่อโปรเจกต์ของคุณ" (เช่น project = "DEV")
    • Events: เลื่อนลงหาหมวด Issue ติ๊กถูกที่ช่อง **Created**, **Updated**, และ **Deleted**
  3. คลิกปุ่มสีน้ำเงินด้านล่างสุด "Create"
5.2 ตั้งค่าเว็บฮุคฝั่ง Google Chat API:
  1. เข้าเว็บ Google Cloud Console เลือกโปรเจกต์งานของคุณ
  2. ค้นหาเปิดบริการ "Google Chat API" แล้วกดเปิดใช้งาน (Enable)
  3. ไปที่เมนู "Configuration" ด้านซ้าย กรอกโปรไฟล์บอตตามชอบ
  4. ที่หัวข้อ **Connection settings** เลือก **"Webhook URL"**
  5. วางที่อยู่อีเมลลิงก์เว็บ Vercel ของคุณ และใส่ต่อท้ายด้วย /api/webhook (เช่น https://my-jira-bot.vercel.app/api/webhook)
  6. คลิกปุ่ม "Save" ล่างสุด
🏁 สเต็ปที่ 6: การเริ่มระบบดึงประวัติครั้งแรก (First Sync)

พิมพ์เปิดลิงก์สั่งงานเพื่อซิงก์ดึงประวัติตั๋วงานเก่าทั้งหมดบน Jira เข้าแดชบอร์ดของคุณเพื่อเตรียมแสดงผล:

https://[ที่อยู่เว็บ_Vercel_ของคุณ]/api/cron/sync-tickets?secret=[รหัส_CRON_SECRET_ของคุณ]

*ตัวอย่าง:* https://my-jira-bot.vercel.app/api/cron/sync-tickets?secret=my_custom_secret_123

เมื่อเบราว์เซอร์แสดงข้อความระบุว่า "sync success" ก็แปลว่าบอตและ Dashboard ของคุณติดตั้งเชื่อมต่อด้วยตนเองเสร็จสมบูรณ์เรียบร้อยแล้วครับ! 🎉
