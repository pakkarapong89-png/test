🔍 1. สิ่งที่ต้องจัดเตรียมก่อนเริ่ม (Prerequisites)

เพื่อรันระบบนี้ให้สมบูรณ์ โปรดเตรียมบัญชีและโปรแกรมต่อไปนี้ให้พร้อม:

•Node.js (เวอร์ชัน 18.0.0 ขึ้นไป)
•บัญชี GitHub และบัญชี Vercel สำหรับอัปโหลดและเปิดใช้งานระบบเว็บฟรี
•บัญชี Supabase สำหรับใช้งานฐานข้อมูลคลาวด์ PostgreSQL ฟรี
•บัญชี Jira Cloud ที่มีสิทธิ์ระดับ Admin
•ห้องกลุ่ม (Space) ในแอป Google Chat ที่คุณต้องการติดตั้งบอท

🗄️ 2. การสร้างและตั้งค่าฐานข้อมูล Supabase

ฐานข้อมูลนี้ใช้จัดเก็บข้อมูลงาน ประวัติ และสถิติร่วมกันระหว่าง Dashboard และบอท
1.ไปที่เว็บไซต์ 👉 Supabase แล้วล็อกอินด้วยบัญชี GitHub ของคุณ (คลิก Start your project -> เลือก Sign in with GitHub)
2.เมื่อเข้าสู่หน้าหลักของแดชบอร์ด ให้คลิกปุ่ม New Project (สีเขียว) และตั้งค่าดังนี้:

- Organization: เลือกชื่อองค์กรบัญชีของคุณ (ปกติคือชื่อบัญชี GitHub)

- Name: ตั้งชื่อโปรเจกต์ฐานข้อมูล เช่น taskyapp-database

- Database Password: กำหนดรหัสผ่านที่ปลอดภัย และจดบันทึกเก็บไว้ใน Notepad ทันที

⚠️ สำคัญมาก: โปรดหลีกเลี่ยงการใช้ตัวอักษรพิเศษอย่าง @, :, /, # ในรหัสผ่านฐานข้อมูล เนื่องจากจะทำให้การเชื่อมโยง URL ฐานข้อมูลล้มเหลว (เกิดข้อผิดพลาด Error 500) แนะนำให้ใช้เฉพาะตัวอักษรพิมพ์ใหญ่ พิมพ์เล็ก และตัวเลขเท่านั้น

- Region: เลื่อนหาและเลือกเป็น Singapore (Southeast Asia) เพื่อความรวดเร็วในการส่งข้อมูลที่ใกล้ประเทศไทยที่สุด

- Pricing Plan: เลือกสิทธิ์แผนงานแบบ Free (ฟรี)

3.คลิกปุ่ม Create new project และรอประมาณ 1 - 2 นาทีจนวงกลมแสดงสถานะกลายเป็นสีเขียวเพื่อเริ่มใช้งานคลาวด์เซิร์ฟเวอร์
4.คลิกปุ่ม Connect (แถบสีเขียวขวาบนสุดของหน้าแดชบอร์ด Supabase)
5.ในหน้าต่าง Connect ที่เด้งขึ้นมา จะเห็นตัวเลือก 5 ปุ่มหลัก ได้แก่: Framework, Server, Direct, ORM, และ MCP (ตามรูปในคู่มือ)

- ให้คลิกเลือกปุ่ม Server (หรือปุ่ม Direct ขึ้นอยู่กับเวอร์ชันแสดงผล)

- จากนั้นในเมนูย่อย ให้ระบุประเภทเป็น Transaction pooler (ห้ามเลือก Session pooler หรือ Direct)

- เลือกรูปแบบแสดงผลแท็บเป็น URI

6.คลิกปุ่ม Copy คัดลอกลิงก์การเชื่อมต่อด้านล่างมาวางลง Notepad ชั่วคราว ซึ่งจะมีหน้าตาดังนี้:

postgresql://postgres.xxxxxx:[YOUR-PASSWORD]@aws-0-ap-southeast-1.pooler.supabase.com:6543/postgres

7.แก้ไขลิงก์ใน Notepad: ลบคำว่า [YOUR-PASSWORD] (รวมเครื่องหมายวงเล็บเหลี่ยมออก) แล้วกรอกรหัสผ่านฐานข้อมูลจริงที่คุณตั้งไว้ในข้อ 2 ลงไปแทนที่ให้เรียบร้อย เช่น:

postgresql://postgres.xxxxxx:MyDatabasePass2026@aws-0-ap-southeast-1.pooler.supabase.com:6543/postgres

นี่คือค่า DATABASE_URL ที่ถูกต้องและพร้อมนำไปกรอก

🔑 3. การเตรียมคีย์เชื่อมต่อของ Jira Cloud (4 ค่า)

ชุดข้อมูลนี้จำเป็นสำหรับให้บอทวิ่งไปอัปเดตงานบนกระดาน Jira ขององค์กรคุณได้สำเร็จ

3.1 JIRA_DOMAIN (โดเมนองค์กรของคุณ)

1.เปิดหน้าแท็บใหม่แล้วล็อกอินเข้าใช้งานกระดาน Jira Cloud ขององค์กรคุณตามปกติ
2.สังเกตที่แถบที่อยู่ด้านบนสุดของเว็บเบราว์เซอร์ (URL bar) เช่น https://mycompany.atlassian.net/jira/your-board
3.คัดลอกเฉพาะข้อความส่วนชื่อโดเมนหลัก (ข้อความที่อยู่หลังเครื่องหมาย https:// จนถึงก่อนตัวเครื่องหมาย / ตัวแรก) เช่น:

mycompany.atlassian.net

จดค่านี้ไว้เป็น JIRA_DOMAIN

3.2 JIRA_EMAIL (อีเมลบัญชีผู้ใช้)

1.คืออีเมลส่วนตัวหรืออีเมลของแอดมินระบบที่คุณใช้สำหรับการล็อกอินเข้าพิมพ์งานและเข้าหน้าตั้งค่า Jira Cloud ขององค์กรคุณ เช่น:

admin@mycompany.com

จดค่านี้ไว้เป็น JIRA_EMAIL

3.3 JIRA_API_TOKEN (รหัสผ่านเข้าถึงข้อมูลผ่านโค้ด)

1.เข้าไปที่หน้าเว็บไซต์จัดการความปลอดภัยของบัญชี Atlassian 👉 Atlassian API Tokens (ล็อกอินด้วยบัญชีอีเมล Jira ตัวเดียวกับข้อ 3.2)
2.คลิกปุ่มสีฟ้า Create API token
3.ระบบจะแสดงหน้าต่างให้ตั้งชื่อเรียก ให้พิมพ์ว่า taskyapp-token เพื่อระบุวัตถุประสงค์
4.คลิกปุ่ม Create
5.รหัส Token ตัวอักษรสุ่มยาวๆ จะแสดงขึ้นมา ให้คลิกปุ่ม Copy และนำรหัสไปวางเก็บใน Notepad ทันที

⚠️ ข้อควรระวังสูงสุด: รหัส Token นี้จะขึ้นแสดงให้คุณเห็นและคัดลอกเพียง "ครั้งเดียว" เท่านั้น หากกดปิดหน้าต่างหรือรีเฟรชหน้าเว็บ รหัสจะซ่อนถาวรและไม่สามารถเรียกดูได้อีก (หากทำรหัสหาย ต้องกดสร้าง Token ชิ้นใหม่ขึ้นมาแทน)

3.4 JIRA_PROJECT_KEY (รหัสย่อของโครงการ)

1.เปิดหน้ากระดานบอร์ด Jira Cloud ของคุณ -> เลือกเข้าไปยังตัวโครงการ (Project) ที่ต้องการเชื่อมต่อใช้งาน
2.สังเกตตัวอักษรย่อภาษาอังกฤษที่อยู่ข้างหน้าตัวเลขรหัสตั๋วงาน เช่น หากตั๋วงานชื่อ KAN-1, KAN-2 ตัวย่อประจำโครงการของคุณคือ KAN
3.หรือคลิกเข้าเมนู Project settings (การตั้งค่าโครงการ) -> ดูตรงหัวข้อ Key เช่น KAN

⚠️ สำคัญ: ต้องกรอกเป็นภาษาอังกฤษตัวพิมพ์ใหญ่ทั้งหมด เช่น KAN ห้ามเขียนสะกดด้วยตัวพิมพ์เล็กหรือตัวผสมเด็ดขาด

💬 4. การสร้าง Webhook ของ Google Chat (ส่วนแจ้งเตือน)

ส่วนนี้ทำหน้าที่เปิดทางให้บอร์ด Jira ยิงรายงานแจ้งเตือนกิจกรรมความเคลื่อนไหวกลับเข้ามาโพสต์ในสเปซแชท
1.เปิดแอป Google Chat (หรือพิมพ์คุยผ่านหัวข้อ Chat ใน Gmail)
2.คลิกเลือกสเปซ (Space) หรือห้องกลุ่มงานที่คุณและทีมงานใช้สนทนากันอยู่
3.คลิกที่ ชื่อหัวข้อห้องแชทด้านบนสุด (ใกล้กับลูกศรชี้ลงด้านบน) -> เมนูย่อยจะแสดงลงมา ให้คลิกเลือกหัวข้อ Apps & integrations (แอปและการผสานรวม)
4.จะมีหน้าต่างป๊อปอัปขึ้นแสดง -> ให้เลือกคลิกแถบเมนูหัวข้อ Webhooks
5.คลิกปุ่ม Add webhook (สีฟ้า)
6.กรอกรายละเอียดดังนี้:

- Name: ตั้งชื่อบอทผู้ทำหน้าที่ประกาศข่าว เช่น taskyapp Notifier

- Avatar URL: (สามารถเว้นว่างได้ หรือนำลิงก์รูปภาพโลโก้ของคุณมาวางเพื่อให้บอทมีรูปโปรไฟล์ที่สวยงาม)

7.คลิกปุ่ม Save (บันทึก)
8.ระบบจะสร้างชื่อ Webhook ขึ้นมาพร้อมกับลิงก์ URL ยาวๆ อยู่ด้านล่างสุด ให้คลิกปุ่ม Copy คัดลอกมาเก็บไว้ใน Notepad

ตัวอย่างลิงก์: https://chat.googleapis.com/v1/spaces/xxxx/messages?key=xxxx&token=xxxx

🤖 5. การสร้าง API Key ของ Google Gemini (สมอง AI)

คีย์นี้ช่วยให้ระบบมีสิทธิ์เรียกใช้สมองปัญญาประดิษฐ์เพื่อช่วยถอดรหัสวิเคราะห์ประโยคคำสั่งภาษาไทย
1.เข้าไปยังเว็บไซต์ 👉 Google AI Studio
2.ลงชื่อเข้าใช้งานด้วยบัญชี Google Account ของคุณ
3.คลิกปุ่มสีฟ้าเด่นชัดชื่อว่า Create API key
4.หน้าต่างป๊อปอัปจะแสดงขึ้นมาให้เลือกโครงการ:

- หากคุณพึ่งเริ่มใช้งาน ให้คลิกเลือกคำสั่ง Create API key in new project เพื่อสั่งสร้างโครงการและสิทธิ์เข้าใช้ AI ฟรีทันที

5.รอให้ระบบประมวลผลหมุนเสร็จ จะปรากฏหน้าต่างรหัสคีย์ขึ้นต้นด้วยคำว่า AIzaSy... ให้คลิกปุ่ม Copy และนำค่ามาเก็บไว้ใน Notepad

⚙️ 6. การรันสคริปต์ตั้งค่าระบบอัตโนมัติ (Local Setup)

สคริปต์นี้จะสร้างไฟล์เชื่อมต่อระบบ (.env) และตั้งค่าสร้างตารางข้อมูล 7 ตารางบน Supabase อัตโนมัติ
1.ดำเนินการแตกไฟล์โปรเจกต์ taskyapp-handoff.zip ลงบนคอมพิวเตอร์ของคุณ
2.เข้าไปยังโฟลเดอร์โครงการ จากนั้นรันไฟล์สคริปต์สำหรับติดตั้ง:

- ระบบ Windows: ดับเบิลคลิกไฟล์ setup.bat

- ระบบ Mac/Linux: เปิด Terminal ในโฟลเดอร์โครงการ รันคำสั่ง chmod +x setup.sh ตามด้วย ./setup.sh

3.สคริปต์จะดาวน์โหลด Dependencies และเปิดหน้าจอถามตอบในหน้าต่าง Terminal
4.กรอกค่าต่างๆ ที่คุณเตรียมไว้ใน Notepad ตอบคำถามให้ครบทั้ง 7 รายการ
5.เมื่อตอบเสร็จ สคริปต์จะบันทึกค่าลงในไฟล์ .env และสร้างตารางข้อมูลในคลาวด์ Supabase ให้ทันที

⑦ 7. การอัปโหลดขึ้น GitHub และการ Deploy บน Vercel

7.1 อัปโหลดขึ้น GitHub (เลือกทำได้ 2 วิธี)

คุณสามารถเลือกอัปโหลดโค้ดโครงการขึ้น GitHub ได้ตามความถนัดดังนี้:

วิธีที่ 1: อัปโหลดผ่านหน้าเว็บเบราว์เซอร์ (ง่ายที่สุด — ไม่ต้องพิมพ์คำสั่ง)

1.เข้าเว็บไซต์ GitHub ล็อกอิน และคลิกปุ่ม New (หรือเครื่องหมาย + ขวาบน) เพื่อสร้าง Repository
2.ตั้งชื่อโครงการ เช่น taskyapp -> เลือกสิทธิ์เป็น Private -> คลิกปุ่ม Create repository
3.ระบบจะเปิดหน้าโปรเจกต์ว่าง ให้มองหาลิงก์คำว่า uploading an existing file (มักจะอยู่บรรทัดล่างๆ ของข้อความแนะนำ) และคลิกเข้าไป
4.เปิดหน้าต่างโฟลเดอร์ taskyapp-handoff/nextjs-app ในคอมพิวเตอร์ของคุณ
5.กดคลุมเลือกไฟล์และโฟลเดอร์ทั้งหมดที่อยู่ข้างในนั้น (เช่น โฟลเดอร์ app/, lib/, components/, และไฟล์ package.json, vercel.json ฯลฯ)

⚠️ ข้อควรระวัง: ห้ามเลือกโฟลเดอร์ node_modules (หากมี) ให้เลือกเฉพาะโค้ดของระบบ

6.ลากไฟล์และโฟลเดอร์ทั้งหมดที่เลือก ไปวางในกรอบอัปโหลดของเว็บ GitHub
7.รอให้หน้าเว็บอัปโหลดเสร็จ จากนั้นเลื่อนลงมาล่างสุด คลิกปุ่มสีเขียว Commit changes เป็นอันเสร็จสิ้น

วิธีที่ 2: อัปโหลดผ่าน Git Command Line (สำหรับสายพิมพ์คำสั่ง)

1.เข้าเว็บ GitHub สร้าง Repository ใหม่แบบ Private และคัดลอกลิงก์ของโปรเจกต์ไว้ เช่น https://github.com/username/taskyapp.git
2.เปิดโปรแกรม Terminal, Command Prompt หรือ Git Bash ในโฟลเดอร์ taskyapp-handoff/nextjs-app บนเครื่องของคุณ
3.พิมพ์รันคำสั่งทีละบรรทัดตามลำดับดังนี้:

```bash

git init

git add .

git commit -m "Initial code commit"

git branch -M main

git remote add origin [วางลิงก์ Repository ของคุณที่นี่]

git push -u origin main

```

4.ระบบจะอัปโหลดไฟล์โค้ดทั้งหมดขึ้น GitHub ทันทีเมื่อพิมพ์เสร็จสเต็ปสุดท้าย

7.2 Deploy บน Vercel

1.ล็อกอิน Vercel ด้วยบัญชี GitHub
2.กดปุ่ม Add New... -> เลือก Project -> ค้นหาและกด Import หลังโปรเจกต์ taskyapp
3.⚠️ ขั้นตอนบังคับ (สำคัญมาก): มองหาหัวข้อ Root Directory -> คลิกปุ่ม Edit -> พิมพ์คำว่า nextjs-app ลงไปและกดบันทึก
4.ไปที่หัวข้อ Environment Variables
5.เปิดไฟล์ .env ในคอมพิวเตอร์ของคุณ คัดลอกเนื้อหาทั้งหมดมาวางในช่องกรอกของ Vercel (ระบบจะตรวจจับและแบ่งค่าให้อัตโนมัติทุกตัวแปร)
6.คลิกปุ่ม Deploy และรอประมาณ 1-2 นาที จากนั้นบันทึก Vercel URL (ลิงก์เว็บย่อของคุณ) เก็บไว้ใช้งาน

🔌 8. การเชื่อมต่อบอทเข้า Google Chat (ต่อตรง HTTPS)

บอทเวอร์ชันนี้รองรับการยิงตรงผ่าน HTTPS URL ทันทีโดยไม่ต้องตั้งค่า Google Apps Script ให้ซับซ้อน
1.เข้าเว็บ 👉 Google Cloud Console เลือก Project ของคุณ
2.ค้นหาคำว่า Google Chat API ในช่องค้นหาด้านบน -> คลิกเปิดใช้งาน (Enable)
3.คลิกแถบเมนู Configuration (การตั้งค่า) ทางด้านซ้าย และระบุค่าต่างๆ ดังนี้:

- App name: taskyapp

- Description: ระบบบอทช่วยเหลือจัดการตั๋วงาน Jira อัจฉริยะ

- Functionality: ติ๊กถูกที่ช่อง [x] Receive 1:1 messages และ [x] Join spaces...

- Connection settings: คลิกเลือก HTTPS URL

- URL: กรอกลิงก์เว็บ Vercel ของคุณตามด้วยเส้นทาง /api/webhook เช่น:

https://your-app.vercel.app/api/webhook

- Visibility: เลือกให้คนในองค์กรใช้งานได้ทุกคน

4.คลิกปุ่ม Save ด้านล่างสุดเพื่อบันทึก
5.เข้าแอป Google Chat -> ไปที่ห้อง Space -> คลิกชื่อห้องด้านบนสุด -> เลือก Apps & integrations -> กดเพิ่มบอทโดยระบุชื่อ taskyapp บอทจะเข้ามาประจำการในห้องทันที

🔗 9. การผูกระบบแจ้งเตือนกลับของ Jira Webhook

ตั้งค่าให้บอร์ดงานใน Jira ยิงโพสต์รายงานกิจกรรมเมื่อมีการแก้ไขสถานะตั๋วงานกลับเข้าแชท Google Chat
1.ล็อกอินระบบ Jira Cloud ของคุณ -> คลิกปุ่มฟันเฟือง Settings ขวาบน -> เลือกหัวข้อ System
2.เลื่อนแถบเมนูด้านซ้ายล่างสุด คลิกเลือกหัวข้อ Webhooks -> กดปุ่ม Create a Webhook
3.กรอกการเชื่อมต่อ:

- URL: กรอกลิงก์ Vercel ของคุณตามด้วย /api/jira/webhook เช่น:

https://your-app.vercel.app/api/jira/webhook

4.เลื่อนลงมาที่หมวด Events ติ๊กเลือกเหตุการณ์ในหัวข้อ Issue ดังนี้:

- [x] created

- [x] updated

5.กดปุ่ม Create บันทึกการอัปเดตข้อมูล

👥 10. การเชื่อมโยงชื่อเล่นของทีมงานในระบบ Dashboard

บอทจะสามารถทำตามคำสั่งมอบงานให้บุคคลได้ เช่น "สั่งให้สมหญิงทำ" ก็ต่อเมื่อคุณได้ผูกบัญชีไว้ล่วงหน้าแล้ว
1.เปิดหน้าเว็บเบราว์เซอร์ไปที่ลิงก์ Vercel ของคุณ และทำการล็อกอินด้วยบัญชีแอดมิน (admin / admin123)
2.คลิกไปที่แถบเมนูจัดการทีม (Team)
3.กดปุ่มเพิ่มสมาชิกใหม่ กรอกชื่อเล่น (Nickname) เป็นภาษาไทย และใช้ระบบค้นหาอัตโนมัติเลือกชื่อ Jira Display Name ของทีมงานคนนั้นให้จับคู่กันอย่างถูกต้อง
4.กดบันทึกข้อมูล ทีมงานทุกคนจะเริ่มพิมพ์มอบงานผ่านแชทบอทได้ทันที

💬 11. คู่มือการสั่งงานบอทด้วยภาษาพูดธรรมชาติ (NLP Commands)

เมื่อต้องการเรียกใช้บอทในห้องกลุ่ม ให้พิมพ์พิมพ์แท็ก @taskyapp นำหน้าเสมอ แต่ถ้าส่งข้อความแบบแชทส่วนตัว (DM) สามารถพิมพ์ข้อความสั่งงานได้ทันทีโดยไม่ต้องแท็ก:

11.1 คำสั่งสร้างตั๋วงานใหม่ (Create Issue)

•การสั่งสร้าง Epic (งานหลัก):

- @taskyapp สร้างงานหลัก พัฒนาดีไซน์หน้าเว็บ

- @taskyapp เปิด Epic เขียนโปรแกรมระบบ API

•การสั่งสร้าง Task ย่อย (ต้องมีคำว่า งานย่อย, จ๊อบ, job หรือ sub-task เสมอ):

- @taskyapp สร้างงานย่อย ออกแบบหน้าจอ Login งานหลักคือ KAN-5 (ผูกเป็นตั๋วย่อยภายใต้ KAN-5)

- @taskyapp สั่งสร้าง job เขียนเชื่อมต่อ API มอบให้ สมหญิง ความสำคัญ สูง

11.2 คำสั่งสร้างหลายงานพร้อมกัน (Bulk Create)

พิมพ์ประโยคคำสั่งเชื่อมด้วยคำว่า "และ" หรือ "and"

•ตัวอย่างการสั่งงาน:

- @taskyapp สร้างงาน ออกแบบหน้าแรก มอบให้ สมใจ และ สร้างงาน เขียนโค้ดหลังบ้าน มอบให้ dev1

11.3 คำสั่งแก้ไขตั๋วงานและเปลี่ยนสถานะ (Update & Transition)

บอทจะระบุพิกัดตั๋วจากรหัสตัวเลขนำหน้าตั๋ว เช่น KAN-3

•แก้ไขรายละเอียด: @taskyapp แก้ไขงาน KAN-3 เปลี่ยนวันกำหนดส่งเป็น 2026-08-01
•เปลี่ยนคนรับผิดชอบ: @taskyapp ย้ายงาน KAN-5 ให้ dev2 รับผิดชอบ
•เปลี่ยนระดับความสำคัญ: @taskyapp เปลี่ยนระดับความสำคัญ KAN-2 เป็น สูงสุด
•เปลี่ยนสถานะงาน:

- @taskyapp ปิดงาน KAN-6 (เปลี่ยนเป็น Done)

- @taskyapp ย้าย KAN-7 ไปหน้ากำลังทำงาน (เปลี่ยนเป็น In Progress)

11.4 คำสั่งเรียกดูสถิติและค้นหาข้อมูล (Query)

•ดูสถิติโครงการ: @taskyapp สรุปภาพรวมงานค้างทั้งหมดให้หน่อย
•ขอสถิติงานเจาะจงคน: @taskyapp สรุปงานค้างของ สมหญิง
•ค้นหางานตามชื่อคีย์เวิร์ด: @taskyapp ค้นหางานคำว่า API

🔧 12. วิธีแก้ไขปัญหาที่พบบ่อย (Troubleshooting)

❌ พิมพ์ส่งหาบอทแล้ว บอทเงียบ ไม่มีการตอบรับใดๆ

•ตรวจสอบการหลับของเซิร์ฟเวอร์ (Vercel Sleep Mode): ในบางโปรเจกต์แบบฟรี หากไม่มีกิจกรรมการพิมพ์คุยนานๆ เซิร์ฟเวอร์จะเข้าสู่โหมดประหยัดพลังงาน การส่งสัญญาณข้อความครั้งแรกอาจเกิด Timeout (ช้ากว่า 10 วินาที)

- วิธีแก้: ทดลองเปิดหน้าเว็บเบราว์เซอร์ Dashboard ของคุณเพื่อกระตุ้นให้ระบบทำงาน จากนั้นลองส่งข้อความทดสอบโดยพิมพ์ @taskyapp ping ดูการตอบสนอง

•ตรวจสอบรูปแบบ Endpoint: ไปที่ Google Cloud -> Google Chat API -> Configuration ตรวจดูว่าช่อง URL กรอกโดเมนเว็บต่อท้ายด้วย /api/webhook ครบถ้วนและเลือกใช้การต่อตรงแบบ HTTPS URL เท่านั้น

❌ บอทตอบกลับในแชทว่าทำสำเร็จ แต่ไม่มีความเคลื่อนไหวใดๆ ในโปรแกรม Jira จริง

•สาเหตุ: การยืนยันสิทธิ์บัญชีล้มเหลว (API Token ของ Jira หรืออีเมลที่กรอกในไฟล์ตั้งค่าทำงานไม่ถูกต้อง)
•วิธีแก้: เปิดเว็บเบราว์เซอร์ไปที่ลิงก์ตรวจสอบสถานะระบบ https://[ลิงก์เว็บของคุณ]/api/health และดูว่าระบบแจ้งว่า Jira มีสถานะเชื่อมต่อเชื่อมโยงสำเร็จหรือไม่ หากขึ้นว่า Error ให้สร้าง API Token ใหม่ใน Atlassian และอัปเดตไฟล์ตั้งค่าบนหน้า Vercel ให้ถูกต้อง
