# คู่มือการย้ายระบบบอทและแดชบอร์ดไปใช้บนบัญชีบริษัทอย่างละเอียด

คู่มือนี้จะอธิบายขั้นตอนการย้ายระบบทั้งหมด (Google Chat Bot, Jira, และ Vercel Server) จากบัญชีส่วนตัว/การศึกษา ไปใช้งานในโดเมนบัญชีของบริษัทอย่างเป็นทางการแบบจับมือทำทีละขั้นตอน

---

## 📋 สรุปข้อมูลที่ต้องใช้ก่อนเริ่ม
1. **บัญชี Google Workspace ของบริษัท** (เช่น `yourname@company.com`) ที่มีสิทธิ์สร้างโครงการบน Google Cloud
2. **บัญชี Jira ของบริษัท** ที่มีสิทธิ์ระดับ Administrator
3. **สิทธิ์ในการเข้าถึง Vercel Dashboard** (หากต้องการอัปเดตค่าเชื่อมต่อ)

---

## 1. การย้ายบอท (Google Chat Bot) ไปยัง Google Workspace ของบริษัท

ขั้นตอนนี้ทำเพื่อสร้างตัวตนบอทตัวใหม่ภายใต้โดเมนบริษัท (เช่น `@Sekloso`) เพื่อให้พนักงานทุกคนค้นหาเจอในแชท

### ขั้นตอนการตั้งค่า:

1. **เข้าสู่ระบบ Google Cloud Console**:
   * เปิดเว็บบราวเซอร์และเข้าลิงก์ [Google Cloud Console](https://console.cloud.google.com/)
   * **สำคัญ**: ตรวจสอบมุมขวาบนให้มั่นใจว่าล็อกอินด้วย **อีเมลของบริษัท** (เช่น `yourname@company.com`)
2. **สร้างโครงการใหม่ (Create Project)**:
   * คลิกที่แถบเลือกโครงการด้านบนสุด (ใกล้โลโก้ Google Cloud)
   * กดปุ่ม **New Project** (โครงการใหม่)
   * ตั้งชื่อโครงการ เช่น `Company-Jira-Bot` แล้วกด **Create**
3. **เปิดใช้งาน Google Chat API**:
   * ที่ช่องค้นหาด้านบนสุด พิมพ์คำว่า `Google Chat API` แล้วกดค้นหา
   * คลิกเข้าไปที่บริการ **Google Chat API**
   * กดปุ่ม **Enable (เปิดใช้งาน)** แล้วรอระบบประมวลผลสักครู่
4. **ตั้งค่าข้อมูลตัวตนบอท (Configuration)**:
   * หลังจากเปิดใช้งานแล้ว ให้คลิกที่แถบ **Configuration** ด้านซ้ายมือ
   * กรอกข้อมูลในแต่ละส่วนดังนี้:
     * **Application Info**:
       * **App name**: ชื่อของบอทที่คุณต้องการให้แสดงในแชท (เช่น `Sekloso`)
       * **Avatar URL**: ใส่ลิงก์รูปภาพโปรไฟล์ของบอท (แนะนำให้อัปโหลดไฟล์รูปขึ้นโฮสต์หรือเว็บฝากรูปสาธารณะ แล้วนำ Direct Link มาวาง)
       * **Description**: คำอธิบายหน้าที่ เช่น `บอทรับคำสั่งย้ายสถานะและอัปเดตข้อมูลตั๋วงาน Jira`
     * **Interactive features**:
       * ติ๊กเลือกช่อง **Receive 1:1 messages** (เปิดให้คนในบริษัทพิมพ์คุยแบบส่วนตัวกับบอทได้)
       * ติ๊กเลือกช่อง **Join spaces and group conversations** (เปิดให้ดึงบอทเข้าทำงานในห้องกลุ่ม/Space ได้)
     * **Connection settings**:
       * เลือกเป็น **HTTP Endpoint URL**
       * นำลิงก์ Webhook ของแดชบอร์ดด้านล่างนี้ไปวางในช่อง:
         `https://nextjs-app-ten-olive-40.vercel.app/api/webhook`
     * **Visibility**:
       * ติ๊กเลือกหัวข้อ **Everyone in your organization** (เพื่อเปิดสิทธิ์ให้ทุกคนในชื่อโดเมนบริษัทแอดใช้งานบอทตัวนี้ได้)
5. **บันทึกการตั้งค่า**:
   * เลื่อนลงไปล่างสุดแล้วกดปุ่ม **Save (บันทึก)**
   * *เรียบร้อยแล้ว! ตอนนี้ทุกคนในบริษัทสามารถค้นหาบอท `@Sekloso` ใน Google Chat ได้แล้ว*

## 2. การเปลี่ยนการเชื่อมต่อ Jira ไปใช้บอร์ดของบริษัท
*(ทำเฉพาะในกรณีที่ต้องการให้ระบบไปอ่าน/เขียนข้อมูลบนโปรเจกต์งานตัวใหม่ของบริษัท)*

### 2.1 สร้าง API Token จากบัญชีบริษัท
1. ล็อกอินเข้าใช้งานหน้าเว็บ [Atlassian Jira](https://www.atlassian.com/) ด้วยบัญชีบริษัทของคุณ
2. เข้าไปที่หน้าสร้างโทเค็นความปลอดภัย: [Atlassian API Tokens](https://id.atlassian.com/manage-profile/security/api-tokens)
3. กดปุ่ม **Create API token**
4. ตั้งชื่อฉลากกำกับ เช่น `Company-Nextjs-Dashboard` แล้วกด **Create**
5. กดปุ่ม **Copy** เพื่อคัดลอกรหัสโทเค็นเก็บไว้ (รหัสนี้จะแสดงเพียงครั้งเดียวเท่านั้น)

### 2.2 อัปเดตตัวแปรใน Vercel Console
1. ล็อกอินเข้าหน้าเว็บ [Vercel Dashboard](https://vercel.com/)
2. คลิกเข้าไปที่โครงการ `nextjs-app`
3. ไปที่แท็บ **Settings** ด้านบนสุด -> เลือกเมนู **Environment Variables** ด้านซ้ายมือ
4. ทำการแก้ไขค่าตัวแปร (Environment Variables) 4 ตัวนี้ให้เป็นของบริษัท:
   * `JIRA_DOMAIN` ➡️ แก้ไขเป็นโดเมนใหม่ของบริษัท (เช่น `mycompany.atlassian.net`)
   * `JIRA_EMAIL` ➡️ แก้ไขเป็นอีเมลบริษัทที่ใช้สร้าง API Token
   * `JIRA_API_TOKEN` ➡️ วางรหัส API Token ตัวใหม่ที่คัดลอกมาจากขั้นตอนก่อนหน้า
   * `JIRA_PROJECT_KEY` ➡️ รหัสคีย์ย่อของโครงการบน Jira ของบริษัท (เช่น `TES`, `PROJ`)
5. กด **Save (บันทึก)** ทุกค่าที่แก้ไข
6. **ทำการดีพลอยใหม่เพื่ออัปเดตค่า**:
   * ไปที่แท็บ **Deployments** ด้านบนสุด
   * คลิกปุ่มจุดสามจุดข้างการดีพลอยล่าสุด แล้วกด **Redeploy** เพื่อให้ระบบดึงค่าตัวแปรบริษัทไปใช้งานจริง

## 3. ติดตั้ง Webhook ตัวรับข้อมูลใหม่บน Jira ของบริษัท
*(เพื่อให้เวลามีคนขยับตั๋วบนเว็บ Jira.com ตรงๆ แล้วมีข้อความแจ้งเตือนเด้งเข้า Google Chat ทันที)*

1. เข้าไปที่หน้าเว็บ Jira Cloud ของบริษัทด้วยบัญชีผู้ดูแลระบบ (Admin)
2. คลิกปุ่มฟันเฟืองที่มุมขวาบน เลือกเมนู **System**
3. เลื่อนแถบเมนูด้านซ้ายล่างสุด คลิกที่หัวข้อ **Webhooks**
4. กดปุ่ม **Create a Webhook** ที่มุมขวาบน
5. กรอกรายละเอียดดังนี้:
   * **Name**: `Google Chat Webhook`
   * **Status**: เลือกเป็น **Enabled**
   * **URL**: นำลิงก์นี้ไปวาง:
     `https://nextjs-app-ten-olive-40.vercel.app/api/jira/webhook`
   * **JQL Query** (ระบุเพื่อกรองรับข้อมูลเฉพาะโปรเจกต์):
     `project = KEY` *(เปลี่ยน KEY เป็นรหัสคีย์โปรเจกต์ของบริษัท เช่น `project = TES`)*
   * **Issue Events** (ติ๊กเลือกเหตุการณ์ที่จะส่งแจ้งเตือน):
     * เลื่อนลงมาที่หัวข้อ **Issue** แล้วติ๊กเลือก 3 ช่องนี้:
       * [x] **created**
       * [x] **updated**
       * [x] **deleted**
6. เลื่อนลงไปล่างสุดแล้วกดปุ่ม **Create** เป็นอันเสร็จสิ้น
