# คู่มือการขอค่า API และการตั้งค่าระบบบอต (ฉบับเข้าใจง่าย ทำตามได้ทันที)

เอกสารนี้รวบรวมทุกขั้นตอนในการขอคีย์ (API Key), รหัสการเข้าถึง (Token) และการตั้งค่าต่าง ๆ ใน Google Chat เพื่อให้บอตเชื่อมต่อกับ Jira และ Google Gemini ได้อย่างสมบูรณ์แบบ

---

## 📌 สิ่งที่เราต้องเตรียมทั้งหมด (มี 4 ส่วนหลัก)
1. **Google Gemini API Key** (เพื่อใช้สมอง AI ในการแปลความหมายคำพูด)
2. **Jira API Credentials** (เพื่อให้บอตเข้าไปดึงหรือจัดการงานใน Jira)
3. **Google Chat Webhook** (สำหรับให้บอตแจ้งเตือนส่งข้อความเข้าห้อง)
4. **Supabase Database URL** (สำหรับจดจำข้อมูลผู้ใช้และตั๋วงาน)

---

## 1. วิธีขอ Google Gemini API Key (สมองของบอต)
*ใช้สำหรับให้ AI เข้าใจภาษาเขียนทั่วไปและแปลงเป็นคำสั่งเพื่อดำเนินการบน Jira*

1. เข้าเว็บไซต์ **[Google AI Studio](https://aistudio.google.com/)**
2. ลงชื่อเข้าใช้งานด้วยบัญชี Google ของคุณ
3. สังเกตที่เมนูด้านซ้ายบน ให้กดปุ่ม **"Get API key"** (ไอคอนรูปกุญแจ)
4. คลิกปุ่ม **"Create API key"** (ปุ่มสีน้ำเงิน)
5. จะมีหน้าต่างแสดงผลขึ้นมา:
   * หากมีโปรเจกต์เดิมอยู่แล้ว สามารถคลิกเลือกโปรเจกต์นั้น แล้วกดสร้างคีย์ได้เลย
   * หากยังไม่มีโปรเจกต์เดิม ให้กด **"Create API key in new project"** เพื่อสร้างโปรเจกต์ใหม่และรับคีย์
6. เมื่อระบบสร้างคีย์เสร็จแล้ว จะมีรหัสคีย์แสดงขึ้นมา ให้กดปุ่ม **"Copy"** เพื่อคัดลอกเก็บไว้ทันที
   > **💡 ข้อสังเกตสำคัญ:** 
   > * ปัจจุบันคีย์เวอร์ชันใหม่ของ Google จะขึ้นต้นด้วยตัวอักษร **`AQ.`** หรือ **`AL.`** (เช่น `AQ.Ab8RN6LEIlsaq...`) ซึ่งเป็นคีย์รูปแบบใหม่ที่มีความปลอดภัยสูงขึ้น สามารถนำค่านี่ไปใส่ในไฟล์ระบบใช้งานได้ทันทีเลยครับ
   > * แนะนำให้กดที่ **ไอคอนรูปคัดลอก (Copy Icon)** เสมอ หลีกเลี่ยงการลากไฮไลต์ตัวอักษรเองบนหน้าจอเพื่อป้องกันการคัดลอกอักขระขาดหาย

## 2. วิธีขอข้อมูลฝั่ง Jira Cloud (หัวใจของการจัดการงาน)
*เราต้องการคัดลอกค่าทั้งหมด 4 ตัว ดังนี้:*

### 2.1 หาค่า `JIRA_DOMAIN` (ลิงก์เว็บ Jira ของเรา)
1. เปิดหน้าเว็บ Jira ของบริษัทคุณขึ้นมา
2. สังเกตที่ช่อง URL (ช่องพิมพ์ชื่อเว็บด้านบนสุดของเบราว์เซอร์)
3. คัดลอกเอาเฉพาะชื่อ Domain หลัก (ห้ามมี `https://` และห้ามมีเครื่องหมาย `/` หรือข้อความต่อท้ายใดๆ)
   * **ตัวอย่าง URL เดิม:** `https://mycompany.atlassian.net/jira/your-board`
   * **ค่าที่จะนำมาใช้:** `mycompany.atlassian.net`

### 2.2 หาค่า `JIRA_EMAIL`
* คือที่อยู่อีเมลที่คุณใช้เข้าสู่ระบบ (Login) ของบัญชี Jira นั้นๆ (เช่น `admin@mycompany.com`)

### 2.3 วิธีขอ `JIRA_API_TOKEN` (รหัสผ่านเพื่อเชื่อมต่อระบบ)
1. เข้าไปที่หน้าการจัดการความปลอดภัยของบัญชี Atlassian: **[id.atlassian.com/manage-profile/security/api-tokens](https://id.atlassian.com/manage-profile/security/api-tokens)**
2. คลิกปุ่ม **"Create API token"**
3. ตั้งชื่อป้ายกำกับสำหรับรหัสนี้ตามต้องการเพื่อกันลืม (เช่น `Jira-GoogleChat-Bot`) แล้วกด **"Create"**
4. หน้าจอจะแสดงรหัสผ่าน API ยาวๆ ที่ขึ้นต้นด้วยตัวอักษร **`ATATT...`** 
5. ให้กดปุ่ม **"Copy"** ไปบันทึกไว้ในที่ปลอดภัยทันที
   *(คำเตือน: รหัสนี้จะแสดงให้ดูเพียงแค่ครั้งเดียวตอนสร้างเสร็จเท่านั้น หากปิดหน้าต่างไปแล้วจะไม่สามารถกดเข้ามาดูคีย์เดิมซ้ำได้อีก ต้องกดสร้างใหม่เท่านั้น)*

### 2.4 หาค่า `JIRA_PROJECT_KEY` (รหัสย่อโปรเจกต์)
1. เปิดหน้าบอร์ดงาน Jira หรือตารางดูงานในโปรเจกต์ของคุณ
2. สังเกตตัวอักษรย่อภาษาอังกฤษที่ระบุอยู่ด้านหน้าหมายเลขตั๋วงาน
3. เช่น หากงานในโปรเจกต์ของคุณมีชื่อว่า `TES-154`, `TES-155` ค่า Project Key ก็คือ **`TES`** (พิมพ์ด้วยตัวอักษรพิมพ์ใหญ่ทั้งหมด)

## 3. วิธีการตั้งค่าใน Google Chat (หน้าต่างคุยกับบอต)
*มี 2 ส่วนหลักๆ คือ การแจ้งเตือนงานเกินกำหนด (ขาออก) และการรับคำสั่งงานแชท (ขาเข้า)*

### 3.1 การขอลิงก์ส่งข้อความแจ้งเตือน (Google Chat Webhook URL)
*ใช้สำหรับส่งรายงานสรุปงานประจำวัน (Daily Summary) หรือส่งแจ้งเตือนด่วนเวลาเจองานล่าช้าเข้าไปที่ห้องแชทของทีม*

1. เปิดแอปพลิเคชัน **Google Chat** ขึ้นมา
2. เข้าไปใน **ห้องแชท (Space)** ที่ต้องการให้บอตส่งรายงานหรือแจ้งเตือนเข้าไป
3. คลิกที่ **ชื่อของห้องแชท** บริเวณด้านบนสุดเพื่อเปิดแถบเมนูตั้งค่า
4. เลือกหัวข้อ **"Apps & integrations" (แอปและการบูรณาการ)** จากนั้นคลิกที่หัวข้อ **"Webhooks" (เว็บฮุค)**
5. คลิกปุ่ม **"Add webhook" (เพิ่มเว็บฮุค)**
6. ตั้งชื่อเว็บฮุคตามต้องการ (เช่น `Tasky Alert Bot`) และใส่ URL ลิงก์รูปภาพที่จะใช้เป็นรูปโปรไฟล์ของบอต (ถ้ามี)
7. กดปุ่ม **"Save" (บันทึก)**
8. ระบบจะให้ลิงก์ URL ยาวๆ มา (ขึ้นต้นด้วย `https://chat.googleapis.com/...`) ให้กดปุ่มคัดลอก (Copy) เก็บไว้ใช้งานในระบบ

### 3.2 การตั้งค่าเพื่อรับส่งคำสั่งกับบอต (Google Chat App)
*ใช้เพื่อให้คนในห้องแชทสามารถพิมพ์ @ชื่อบอต เพื่อสั่งงานหรือขอดูงานจากในห้องแชทได้*

1. เข้าไปที่หน้าเว็บ **[Google Cloud Console](https://console.cloud.google.com/)**
2. เลือกโปรเจกต์ของคุณ (หรือกดสร้างโปรเจกต์ใหม่ที่เมนูด้านบน)
3. ค้นหาคำว่า **"Google Chat API"** ในช่องค้นหาด้านบนสุด แล้วกดคลิกเข้าสู่หน้าบริการ จากนั้นคลิกปุ่ม **"Enable"** (เปิดใช้งาน)
4. เมื่อเปิดใช้งานเสร็จเรียบร้อย ให้มองแถบเมนูด้านซ้ายและคลิกเลือกหัวข้อ **"Configuration" (การกำหนดค่า)**
5. ทำการกรอกข้อมูลของบอตที่จะให้แสดงในแชท:
   * **App name:** ชื่อบอตที่จะให้แสดงเวลาคน @หา (เช่น `TaskyBot`)
   * **Avatar URL:** ลิงก์รูปโปรไฟล์บอต
   * **Description:** คำอธิบายหน้าที่ของบอตสั้นๆ (เช่น บอตรวมงาน Jira และแจ้งเตือนทีม)
6. เลื่อนลงมาในหัวข้อ **Interactive Connection Settings** ให้คลิกเปิดการทำงานของช่องทาง **"Connection settings"**
7. เลือกติ๊กที่หัวข้อ **"Webhook URL" (หรือ HTTPS Endpoint)**
8. นำลิงก์ URL ของแอปที่ได้จากการ Deploy บน **Vercel** มากอกลงในช่อง โดยพิมพ์ต่อท้ายลิงก์ด้วยคำว่า `/api/webhook`
   * **ตัวอย่างลิงก์ Vercel:** `https://nextjs-app-ten-olive-40.vercel.app`
   * **ลิงก์ที่ต้องกรอกในช่อง:** `https://nextjs-app-ten-olive-40.vercel.app/api/webhook`
9. เลื่อนลงมาล่างสุดในหัวข้อ **Visibility** ให้เลือกติ๊กเลือกหัวข้อ **"Only people and groups in my domain..."** (เพื่อให้เรียกใช้งานบอตได้เฉพาะบุคคลในองค์กร/โดเมนของคุณเท่านั้น)
10. กดปุ่ม **"Save" (บันทึก)** เป็นอันเสร็จสิ้นขั้นตอนในฝั่ง Google Chat App

## 4. วิธีขอ Supabase Database URL (ระบบฐานข้อมูล)
*ฐานข้อมูล PostgreSQL สำหรับเก็บข้อมูลตั๋วงาน, รายชื่อทีม, และบันทึกกิจกรรมกิจกรรม (Activity logs)*

1. เข้าเว็บไซต์ **[Supabase.com](https://supabase.com/)** สมัครสมาชิกและคลิกสร้างโปรเจกต์ใหม่ (**New Project**)
2. ตั้งชื่อโปรเจกต์และกำหนดรหัสผ่านฐานข้อมูล (Database Password) ให้เรียบร้อย
3. เมื่อโปรเจกต์พร้อมใช้ ให้ไปที่เมนู **Settings** (ไอคอนรูปฟันเฟืองด้านซ้ายล่าง) -> เลือกหัวข้อ **Database**
4. เลื่อนลงมาหาหัวข้อ **Connection string** และกดคลิกเลือกแท็บ **URI**
5. คัดลอกลิงก์ที่ขึ้นต้นด้วย `postgres://...` 
6. นำลิงก์ที่ได้มาแก้ไขข้อความตรงช่อง `[YOUR-PASSWORD]` ให้กลายเป็นรหัสผ่านจริงที่คุณเพิ่งตั้งไปในขั้นตอนสร้างโปรเจกต์
   * **ตัวอย่างลิงก์สำเร็จรูปที่พร้อมใช้:** `DATABASE_URL="postgres://postgres.abc:MyPassword123@aws-0-ap-southeast-1.pooler.supabase.com:6543/postgres"`

## 🛠️ ขั้นตอนสุดท้าย: นำค่าไปใส่ในระบบ
นำค่าคีย์และรหัสต่างๆ ที่รวบรวมได้จากด้านบน ไปกรอกใส่ในไฟล์คอนฟิก **`.env`** บนเครื่องคอมพิวเตอร์ หรือนำไปใส่ในหัวข้อ **Environment Variables บนหน้าแดชบอร์ด Vercel** (สำหรับตัวเว็บที่รันออนไลน์):

```env
DATABASE_URL="ใส่ค่าเชื่อมต่อ Supabase จากข้อ 4"

JIRA_DOMAIN="ใส่ค่าจากข้อ 2.1 (เช่น mycompany.atlassian.net)"
JIRA_EMAIL="ใส่ค่าจากข้อ 2.2"
JIRA_API_TOKEN="ใส่ค่าจากข้อ 2.3 (ขึ้นต้นด้วย ATATT...)"
JIRA_PROJECT_KEY="ใส่ค่าจากข้อ 2.4 (เช่น TES)"

GOOGLE_CHAT_WEBHOOK_URL="ใส่ค่าจากข้อ 3.1"

LLM_PROVIDER="gemini"
GEMINI_API_KEY="ใส่คีย์จากข้อ 1 (คีย์เวอร์ชันใหม่ขึ้นต้นด้วย AQ. หรือ AL.)"
GEMINI_MODEL="gemini-3.5-flash"
BOT_NAME="ตั้งชื่อบอตของคุณ"

SESSION_SECRET="พิมพ์คีย์สุ่มยาวๆ เพื่อความปลอดภัยของเซสชันล็อกอิน"
CRON_SECRET="พิมพ์คีย์สุ่มยาวๆ เพื่อความปลอดภัยในการเรียกฟังก์ชันตั้งเวลา (Cron)"
```
