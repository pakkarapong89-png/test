# คู่มือการตั้งค่าบอตและอัปโหลดระบบใช้งานจริง (ฉบับสมบูรณ์ทำตามได้ทันที)

เอกสารนี้รวบรวมทุกขั้นตอนตั้งแต่การขอคีย์เชื่อมต่อระบบ (API Credentials), การนำโค้ดขึ้นระบบคลาวด์ (Deploy to Vercel), ไปจนถึงการตั้งค่าห้องแชทและ Webhooks เพื่อให้ระบบบอต Google Chat + Jira + Dashboard สามารถทำงานประสานกันได้อย่างสมบูรณ์แบบ

---

## 📌 แผนภาพการทำงานของระบบ
```
[ Google Chat ] <---> ( บอตหลังบ้านบน Vercel ) <---> [ Jira Cloud ]
                             |
                             +---> [ ฐานข้อมูล Supabase ]
```

---

## 1. วิธีขอคีย์เชื่อมต่อระบบ (API Credentials)
*เป็นขั้นตอนการเตรียมกุญแจสำคัญ 4 ดอกก่อนเริ่มระบบ*

### 🔑 1.1 ขอ Google Gemini API Key (สมอง AI แปลข้อความ)
1. เข้าไปที่หน้าเว็บ **[Google AI Studio](https://aistudio.google.com/)**
2. ลงชื่อเข้าใช้งานด้วยบัญชี Google
3. คลิกปุ่ม **"Get API key"** (ไอคอนกุญแจซ้ายบน)
4. คลิกปุ่มสีน้ำเงิน **"Create API key"** 
5. เลือกว่าจะสร้างในโปรเจกต์เดิม หรือคลิกสร้างโปรเจกต์ใหม่ (**Create API key in new project**)
6. เมื่อระบบสร้างคีย์เสร็จ ให้กด **Copy** บันทึกรหัสเก็บไว้ทันที
   > **💡 จุดสังเกตคีย์แบบใหม่ (ปี 2026):** คีย์เวอร์ชันใหม่ของกูเกิลจะขึ้นต้นด้วยอักษร **`AQ.`** หรือ **`AL.`** (เช่น `AQ.Ab8RN6L...`) ซึ่งสามารถคัดลอกและนำไปใช้รันระบบจริงได้ทันทีเลยครับ

---

### 🔑 1.2 ขอข้อมูลและสิทธิ์เชื่อมต่อ Jira Cloud
เราต้องเตรียมข้อมูลทั้งหมด 4 ค่าดังนี้:
1. **JIRA_DOMAIN (ชื่อโดเมน Jira):** ให้เปิดเว็บ Jira ของคุณ แล้วคัดลอกเอาเฉพาะชื่อเว็บหลักมาใช้ **(ห้ามใส่ https:// และห้ามมีตัวอักษร / ต่อท้าย)**
   * *ตัวอย่าง:* URL คือ `https://myteam.atlassian.net/jira/projects` ให้คัดลอกเอาเฉพาะ `myteam.atlassian.net`
2. **JIRA_EMAIL:** อีเมลที่คุณใช้ Login เข้าไปทำงานบนบอร์ด Jira
3. **JIRA_API_TOKEN (รหัสผ่านเชื่อมต่อ):**
   * ล็อกอินแล้วเข้าลิงก์นี้: **[id.atlassian.com/manage-profile/security/api-tokens](https://id.atlassian.com/manage-profile/security/api-tokens)**
   * คลิกปุ่ม **"Create API token"**
   * ตั้งชื่อป้ายกำกับเพื่อกันลืม (เช่น `Jira-GoogleChat-Bot`) แล้วกด **"Create"**
   * ระบบจะแสดงรหัสขึ้นต้นด้วย **`ATATT...`** ให้กด **Copy** บันทึกไว้ในที่ปลอดภัยทันที *(รหัสจะแสดงแค่ครั้งเดียวเท่านั้น)*
4. **JIRA_PROJECT_KEY (รหัสย่อโปรเจกต์):**
   * สังเกตตัวอักษรย่อภาษาอังกฤษที่อยู่ด้านหน้าหมายเลขตั๋วงานบนบอร์ด เช่น หากตั๋วงานมีรหัสย่อเป็น `TES-154` ค่า Project Key ของคุณก็คือ **`TES`** (พิมพ์ด้วยตัวอักษรพิมพ์ใหญ่)

---

### 🔑 1.3 ขอลิงก์ส่งการแจ้งเตือน (Google Chat Webhook URL)
*สำหรับให้ระบบส่งรายงานสรุปงานประจำวัน (Daily Summary) และแจ้งเตือนตั๋วงานค้างเข้าห้องแชทอัตโนมัติ*
1. เปิดห้องแชท (Space) ใน Google Chat ที่คุณทำงานร่วมกับทีม
2. คลิกเลือกที่ **ชื่อห้องแชท** บริเวณด้านบนสุดเพื่อเปิดหน้าต่างตั้งค่า
3. คลิกเลือกเมนู **"Apps & integrations" (แอปและการบูรณาการ)** -> คลิกหัวข้อ **"Webhooks" (เว็บฮุค)**
4. คลิกปุ่ม **"Add webhook" (เพิ่มเว็บฮุค)**
5. ตั้งชื่อโปรไฟล์บอต (เช่น `Tasky Alert Bot`) และใส่รูปภาพโปรไฟล์บอตตามชอบ
6. กดปุ่ม **"Save" (บันทึก)** แล้วกด **Copy** ลิงก์ URL ที่ระบบให้มา (จะขึ้นต้นด้วย `https://chat.googleapis.com/...`)

---

### 🔑 1.4 ขอลิงก์เชื่อมฐานข้อมูล Supabase Database
*ระบบนี้ใช้ Supabase เพื่อเก็บบันทึกประวัติ รายชื่อทีม และตั๋วงานไปแสดงผลบนหน้า Dashboard*
1. สมัครใช้งานและเข้าสู่ระบบที่ **[Supabase.com](https://supabase.com/)**
2. กดสร้างโปรเจกต์ใหม่ (**New Project**) พร้อมกำหนด Database Password ให้แข็งแรง
3. มองหาแถบเมนูด้านบนสุดของโปรเจกต์ แล้วคลิกปุ่มสีเขียว **"Connect"**
4. ในหน้าต่างที่เด้งขึ้นมา ให้เลือกแท็บ **"Direct"** ด้านบนสุด
5. สังเกตหัวข้อ **Connection Method** ให้เลือกตัวเลือก **"Transaction pooler"**
6. ที่ช่อง **Type** ด้านล่าง ให้เลือกเป็น **"URI"**
7. กดปุ่มคัดลอก (Copy) ลิงก์ที่ขึ้นต้นด้วย `postgres://...`
8. แก้ไขลิงก์โดยนำรหัสผ่านฐานข้อมูลที่คุณตั้งไว้ไปแทนที่คำว่า `[YOUR-PASSWORD]`
   * *ตัวอย่างลิงก์ที่พร้อมใช้:* `postgres://postgres.abc:Password123@aws-0-ap-southeast-1.pooler.supabase.com:6543/postgres`

## 2. ขั้นตอนการนำโค้ดขึ้นระบบและรันใช้งานจริง (Deploy to Vercel)
เมื่อเตรียมกุญแจสำคัญครบ 4 ข้อแล้ว ขั้นตอนต่อไปคือการนำโค้ดระบบขึ้นโฮสติ้งออนไลน์ผ่าน GitHub และ Vercel:

### 🚀 2.1 อัปโหลดโค้ดขึ้น GitHub
1. สมัครบัญชีใช้งานที่ **[GitHub.com](https://github.com/)**
2. กดสร้างคลังเก็บโค้ดใหม่ (**Create a new repository**) ตั้งชื่อโปรเจกต์ เช่น `jira-googlechat-bot` 
3. ตั้งสถานะโปรเจกต์เป็น **Private (ส่วนตัว)** เสมอ เพื่อความปลอดภัย
4. ดำเนินการ Commit โค้ดในโปรเจกต์ทั้งหมดแล้ว Push (อัปโหลด) ขึ้น GitHub ไปที่ Repository ดังกล่าว
   > **⚠️ คำเตือนสำคัญ:** ห้ามนำค่าคีย์จริง หรือไฟล์ `.env` อัปโหลดลง GitHub เด็ดขาด (ระบบนี้มีไฟล์ `.gitignore` บล็อกไฟล์เหล่านี้ให้อยู่แล้วเพื่อความปลอดภัย)

### 🚀 2.2 เชื่อมต่อโปรเจกต์กับ Vercel
1. สมัครใช้งานที่ **[Vercel.com](https://vercel.com/)** โดยล็อกอินผ่านบัญชี GitHub ของคุณ
2. คลิกปุ่ม **"Add New"** -> เลือกหัวข้อ **"Project"**
3. ระบบจะให้เลือกโปรเจกต์จาก GitHub ให้คลิกปุ่ม **"Import"** หลังชื่อ Repository `jira-googlechat-bot` ที่เราเพิ่งอัปโหลดขึ้นไป
4. ในหน้าจอตั้งค่าตัวแปรระบบ (ก่อนกด Deploy) ให้เลื่อนลงมาหาหัวข้อ **Environment Variables**
5. นำค่าคีย์ทั้งหมดที่ได้เตรียมไว้ในขั้นตอนที่ 1 ไปเพิ่มเข้าไปในตารางทีละตัว ดังนี้:

| ชื่อตัวแปร (Key) | วิธีใส่ค่า (Value) |
| --- | --- |
| `DATABASE_URL` | ลิงก์เชื่อม Supabase ที่ใส่รหัสผ่านแล้ว (จากข้อ 1.4) |
| `JIRA_DOMAIN` | ชื่อโดเมน Jira เช่น `myteam.atlassian.net` (จากข้อ 1.2) |
| `JIRA_EMAIL` | อีเมลบัญชี Jira ของท่าน (จากข้อ 1.2) |
| `JIRA_API_TOKEN` | รหัส API Token ขึ้นต้นด้วย ATATT... (จากข้อ 1.2) |
| `JIRA_PROJECT_KEY` | รหัสย่อโปรเจกต์ เช่น TES (จากข้อ 1.2) |
| `GOOGLE_CHAT_WEBHOOK_URL` | ลิงก์แจ้งเตือนของห้องแชท (จากข้อ 1.3) |
| `LLM_PROVIDER` | พิมพ์คำว่า `gemini` |
| `GEMINI_API_KEY` | คีย์ API ขึ้นต้นด้วย AQ. หรือ AL. (จากข้อ 1.1) |
| `GEMINI_MODEL` | พิมพ์คำว่า `gemini-3.5-flash` |
| `BOT_NAME` | ตั้งชื่อบอตของคุณ เช่น `TaskyBot` |
| `SESSION_SECRET` | พิมพ์ตัวอักษรและตัวเลขสุ่มยาวๆ เพื่อความปลอดภัยล็อกอิน |
| `CRON_SECRET` | พิมพ์ตัวอักษรและตัวเลขสุ่มยาวๆ สำหรับทำระบบตั้งเวลาอัตโนมัติ |

6. ตรวจสอบชื่อตัวแปรและข้อมูลว่าถูกต้องครบถ้วน จากนั้นคลิกปุ่ม **"Deploy"**
7. รอประมาณ 1-2 นาที เมื่อระบบประมวลผลเสร็จสิ้น Vercel จะแสดงหน้าจอฉลองความสำเร็จ พร้อมลิงก์ที่อยู่เว็บสาธารณะของคุณมาให้ (เช่น `https://jira-googlechat-bot.vercel.app`) ให้คัดลอกที่อยู่เว็บนี้เก็บไว้

## 3. ขั้นตอนการตั้งค่า Webhook หลังการ Deploy
*ขั้นตอนนี้สำคัญมาก! เพื่อให้ระบบ Jira และห้องแชท Google Chat คุยโต้ตอบส่งข้อมูลกลับมาหาเซิร์ฟเวอร์บอตได้*

### 🔌 3.1 ตั้งค่าเว็บฮุคในฝั่ง Jira Cloud (เพื่อให้ตั๋วซิงก์กลับเข้าแดชบอร์ดอัตโนมัติ)
ทุกครั้งที่มีคนสร้างงาน แก้ไขงาน หรือกดปิดงานใน Jira ตัวระบบ Jira จะต้องยิงข่าวสารมาอัปเดตหน้า Dashboard ของเราทันที โดยมีวิธีการตั้งค่าดังนี้:
1. เปิดเว็บ Jira ของคุณ และตรวจสอบว่าคุณเป็น Admin โปรเจกต์
2. เข้าไปที่หน้าการตั้งค่าเว็บฮุคผ่านลิงก์ทางลัดนี้: **`https://[JIRA_DOMAIN_ของคุณ]/plugins/servlet/webhooks`**
   *(เช่น `https://myteam.atlassian.net/plugins/servlet/webhooks`)*
3. คลิกปุ่ม **"Create a Webhook"** ที่ขวาบน
4. กำหนดรายละเอียดเว็บฮุคดังนี้:
   * **Name:** ชื่อเว็บฮุค (เช่น `Sync to Dashboard Web`)
   * **URL:** นำที่อยู่ลิงก์เว็บ Vercel ของคุณมากรอก และต่อท้ายด้วย `/api/jira/webhook` เสมอ
     * *ตัวอย่าง:* `https://jira-googlechat-bot.vercel.app/api/jira/webhook`
   * **Scope:** ป้อนคำสั่งเลือกเจาะจงเฉพาะโครงการของคุณได้ที่ช่อง JQL เช่น `project = "TES"` (เปลี่ยน TES เป็นรหัสโปรเจกต์ของคุณ)
   * **Events (กิจกรรมที่จะส่งสัญญาณ):** เลื่อนลงมาที่หัวข้อ **Issue** แล้วติ๊กเลือก 3 ช่องนี้:
     * **Created** (เมื่อมีการสร้างงาน)
     * **Updated** (เมื่อมีการแก้ไขงานหรืออัปเดตสถานะ)
     * **Deleted** (เมื่อมีการลบงาน)
5. เลื่อนลงมาล่างสุดแล้วกดปุ่ม **"Create"** บันทึกค่าข้อมูล

---

### 🔌 3.2 ตั้งค่าแอปพลิเคชันห้องแชท (Google Chat App)
ขั้นตอนเพื่อให้เราสามารถพิมพ์ `@ชื่อบอต` สั่งงานในห้องแชทได้:
1. เข้าเว็บ **[Google Cloud Console](https://console.cloud.google.com/)**
2. เลือกโปรเจกต์ของคุณให้ถูกต้องที่แถบด้านบน
3. ค้นหาบริการคำว่า **"Google Chat API"** ในช่องค้นหาด้านบนสุด แล้วกดคลิกเข้าสู่หน้าบริการ จากนั้นคลิกปุ่ม **"Enable"** (เปิดใช้งาน)
4. เมื่อเปิดใช้งานเสร็จเรียบร้อย ให้มองแถบเมนูด้านซ้ายและคลิกเลือกหัวข้อ **"Configuration" (การกำหนดค่า)**
5. ทำการกรอกข้อมูลของบอตที่จะให้แสดงในแชท:
   * **App name:** ชื่อบอตที่จะให้แสดงเวลาคน @หา (เช่น `TaskyBot`)
   * **Avatar URL:** ลิงก์รูปโปรไฟล์บอต
   * **Description:** คำอธิบายหน้าที่ของบอตสั้นๆ (เช่น บอตรวมงาน Jira และแจ้งเตือนทีม)
6. เลื่อนลงมาในหัวข้อ **Interactive Connection Settings** ให้คลิกเปิดการทำงานของช่องทาง **"Connection settings"**
7. เลือกติ๊กที่หัวข้อ **"Webhook URL" (หรือ HTTPS Endpoint)**
8. นำที่อยู่ลิงก์เว็บ Vercel ของคุณมากรอก และต่อท้ายด้วย `/api/webhook` เสมอ
   * *ตัวอย่าง:* `https://jira-googlechat-bot.vercel.app/api/webhook`
9. ส่วนหัวข้อ **Visibility** ด้านล่างสุด ให้เลือกติ๊กเลือกหัวข้อ **"Only people and groups in my domain..."** (เพื่อให้เรียกใช้งานบอตได้เฉพาะบุคคลในโดเมนเมลองค์กรของคุณเท่านั้น)
10. กดปุ่ม **"Save" (บันทึก)** ข้อมูลทั้งหมดเป็นอันเสร็จสมบูรณ์

## 4. เริ่มต้นการใช้งานและการดึงตั๋วงานครั้งแรก (First Sync)
เพื่อให้ฐานข้อมูล Supabase และหน้าเว็บแสดงผลมีประวัติงานเก่าและใหม่ของ Jira ทั้งหมด:
1. เปิดหน้าเว็บแดชบอร์ดที่คุณพึ่ง Deploy เสร็จ (เช่น `https://jira-googlechat-bot.vercel.app/dashboard`)
2. สมัครสมาชิกบัญชี Admin แรกของระบบ และ Login เข้าใช้งาน
3. ระบบจะยังว่างเปล่าไม่มีข้อมูลงาน ให้คุณเปิดเบราว์เซอร์ไปที่ลิงก์ดึงข้อมูลนี้:
   * **รูปแบบลิงก์:** `https://[ชื่อเว็บคลาวด์ของคุณ]/api/cron/sync-tickets?secret=[ค่า_CRON_SECRET_ของคุณ]`
   * *ตัวอย่าง:* `https://jira-googlechat-bot.vercel.app/api/cron/sync-tickets?secret=my_cron_secret_123`
4. หน้าจอจะแสดงข้อความว่า `sync success` เป็นสัญญาณว่าระบบดึงตั๋วงานและข้อมูลทีมพัฒนาทั้งหมดมาจัดเก็บในระบบเรียบร้อยแล้ว
5. กลับไปเปิดหน้า Dashboard งานจะขึ้นแสดงผลครบถ้วน

## ⏰ 5. การตั้งค่าสรุปรายงานประจำวันอัตโนมัติ (Vercel Cron)
ระบบนี้ได้รับการเขียนโปรแกรมและโครงสร้างไว้ในไฟล์ `vercel.json` เรียบร้อยแล้ว เมื่อนำขึ้นรันบน Vercel ตัวเว็บ Vercel จะตั้งเวลารันให้อัตโนมัติ:
* **การซิงก์ข้อมูลงานค้างอัตโนมัติ:** รันทุกชั่วโมงเพื่อตรวจจับการเปลี่ยนแปลงจาก Jira มาอัปเดตฐานข้อมูลแดชบอร์ด
* **การสรุปรายงานประจำวัน (Daily Summary):** รันทุกวันเวลา 18:00 น. (หรือปรับเปลี่ยนเวลาในไฟล์ `vercel.json` ได้ตามต้องการ) เพื่อคำนวณและสรุปงานส่งแจ้งเตือนเข้าไปที่ช่อง Google Chat Space
