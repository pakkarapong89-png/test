คู่มือการขอค่า API และการตั้งค่าระบบบอต

ฉบับอัปเดตล่าสุดปี 2026 — เรียบง่าย อ่านแล้วทำตามได้ทันที ไม่สับสน
📌 สารบัญขั้นตอนการตั้งค่า
  • 1. การขอ Google Gemini API Key (คีย์สมอง AI)
  • 2. การขอข้อมูลและเชื่อมต่อฝั่ง Jira Cloud
  • 3. การกำหนดค่าในห้องแชท Google Chat (Webhook & Chat App)
  • 4. การสร้างและดึงค่า Supabase Database URL
  • 5. สรุปการนำค่าไปใส่ในไฟล์ตั้งค่าระบบ (.env & Vercel)

1. วิธีขอ Google Gemini API Key (สมองของบอต)

คีย์นี้จะช่วยเชื่อมระบบบอตเข้ากับโมเดลปัญญาประดิษฐ์ เพื่อช่วยแปลงภาษาพูดของคนในแชทให้เป็นคำสั่งจัดการงานใน Jira ได้

  1. เปิดหน้าเว็บเบราว์เซอร์แล้วเข้าสู่ลิงก์: Google AI Studio
  2. ล็อกอินด้วยบัญชี Google ของคุณให้เรียบร้อย
  3. มองหาปุ่มเมนูทางซ้ายมือ แล้วคลิกหัวข้อ "Get API key" (ไอคอนรูปกุญแจ)
  4. คลิกปุ่มสีน้ำเงินที่เขียนว่า "Create API key"
  5. ระบบจะให้คุณเลือกโปรเจกต์:
    • หากพึ่งเคยใช้งานเป็นครั้งแรก ให้คลิกปุ่ม "Create API key in new project"
    • หากมีโปรเจกต์ที่สร้างไว้แล้วในเครื่อง ให้คลิกเลือกโปรเจกต์นั้น แล้วกดดำเนินการต่อ
  6. เมื่อระบบสร้างคีย์สำเร็จ จะแสดงแถบยาวๆ ขึ้นมา ให้คลิกปุ่ม "Copy" เพื่อบันทึกค่าไว้ในเครื่องทันที
💡 อัปเดตข้อมูลเรื่องคีย์ Gemini แบบใหม่ (ปี 2026):
ปัจจุบัน Google เปลี่ยนรูปแบบการออกคีย์ใหม่ เพื่อให้ปลอดภัยขึ้น คีย์ของท่านจะขึ้นต้นด้วยตัวอักษร "AQ." หรือ "AL." เสมอ (เช่น AQ.Ab8RN6LEI...) ท่านสามารถคัดลอกค่านี้ไปใช้ได้ทันที ไม่ต้องตกใจว่าทำไมรูปแบบคีย์ถึงต่างจากเดิมครับ

2. วิธีขอข้อมูลและคีย์ฝั่ง Jira Cloud

เพื่อให้ระบบบอตของคุณสามารถเข้าไปอัปเดตงาน เพิ่มงาน หรือดึงข้อมูลงานในบอร์ด Jira มาแสดงผลได้ เราจำเป็นต้องเตรียมค่า 4 ตัวนี้:

2.1 หาค่า JIRA_DOMAIN (ที่อยู่เว็บบอร์ด Jira)

  • ให้ไปที่หน้าเว็บ Jira ที่คุณทำงานตามปกติ แล้วสังเกตช่องใส่ URL ด้านบนสุดของเบราว์เซอร์
  • คัดลอกเอาเฉพาะชื่อ Domain หลักเท่านั้น (ห้ามใส่ https:// และห้ามใส่เครื่องหมายสแลช / ต่อท้าย)
  • ตัวอย่าง URL จริง: https://mycompany.atlassian.net/jira/your-board
  • ค่าที่ต้องคัดลอกมาใช้: mycompany.atlassian.net

2.2 หาค่า JIRA_EMAIL

  • คือที่อยู่อีเมลจริงที่คุณใช้สมัครหรือล็อกอินเข้าใช้บอร์ดงาน Jira ดังกล่าว (เช่น admin@mycompany.com)

2.3 วิธีสร้าง JIRA_API_TOKEN

  1. ล็อกอินและเข้าหน้าตั้งค่าความปลอดภัยของ Atlassian ผ่านลิงก์นี้: Atlassian API Tokens
  2. คลิกปุ่มสีน้ำเงิน "Create API token"
  3. ตั้งชื่อป้ายกำกับเพื่อกันลืม (เช่น Jira-GoogleChat-Bot) จากนั้นกดปุ่ม "Create"
  4. ระบบจะแสดงรหัส Token ยาวๆ ขึ้นต้นด้วยตัวอักษร "ATATT..."
  5. ให้กดปุ่ม "Copy" แล้วรีบนำไปจดบันทึกไว้ในที่ปลอดภัยทันที
    *ข้อควรระวัง: รหัสนี้จะแสดงให้ดูเพียงแค่ครั้งเดียวตอนสร้างเสร็จเท่านั้น หากปิดหน้าต่างนี้ไปจะไม่สามารถกลับมาเปิดดูคีย์เดิมซ้ำได้อีก*

2.4 หาค่า JIRA_PROJECT_KEY

  • ให้เปิดดูตั๋วงานในบอร์ด Jira ของทีมคุณ
  • ค่านี้คือตัวอักษรย่อภาษาอังกฤษที่อยู่ด้านหน้ารหัสตั๋วงาน เช่น ถ้างานในบอร์ดคือ TES-154, TES-155 รหัสโปรเจกต์ของคุณคือ TES (พิมพ์ด้วยตัวพิมพ์ใหญ่ทั้งหมด)

3. วิธีการกำหนดค่าและตั้งค่าใน Google Chat

เพื่อให้แชทของทีมส่งสัญญาณและพูดคุยโต้ตอบกับบอตได้ จะแบ่งออกเป็น 2 ขา ดังนี้:

3.1 การแจ้งเตือนขาออก (Google Chat Webhook URL)

ใช้สำหรับให้บอตสามารถส่งข้อความแจ้งสรุปงานประจำวัน (Daily Summary) หรือยิงข้อความเตือนเมื่อมีตั๋วงานค้างเข้าห้องแชทเองอัตโนมัติ

  1. เปิดโปรแกรม Google Chat หรือ Google Chat ใน Gmail
  2. เข้าไปที่ห้องแชทของทีม (Space) ที่ต้องการให้บอตส่งข้อความเข้ามา
  3. คลิกที่แถบ **ชื่อห้องแชท** ด้านบนสุด เพื่อเปิดแถบเมนูเสริม
  4. เลือกเมนู **"Apps & integrations" (แอปและการบูรณาการ)** -> คลิกหัวข้อ **"Webhooks" (เว็บฮุค)**
  5. คลิกเลือกปุ่ม **"Add webhook" (เพิ่มเว็บฮุค)**
  6. ตั้งชื่อเว็บฮุค (เช่น Tasky Alert Bot) และใส่ภาพโปรไฟล์ของบอตที่คุณชื่นชอบ
  7. กดปุ่ม **"Save" (บันทึก)** จากนั้นระบบจะสร้างลิงก์ให้ ให้กดคลิก **คัดลอกค่าลิงก์ URL** ยาวๆ ที่ได้รับมาเก็บไว้ใช้งาน

3.2 การรับคำสั่งขาเข้า (Google Chat App)

ใช้สำหรับการตั้งค่าแอปพลิเคชัน เพื่อให้คนในองค์กรพิมพ์ @ชื่อบอต แล้วส่งข้อความคุยกับบอตได้โดยตรง

  1. ล็อกอินเข้าใช้งานที่เว็บ Google Cloud Console
  2. เลือกโปรเจกต์ระบบบอตของคุณที่แถบเมนูด้านบนสุด
  3. ค้นหาบริการคำว่า "Google Chat API" ในช่องค้นหาด้านบน และคลิกเข้าไปกดปุ่ม "Enable" (เปิดใช้งาน)
  4. เมื่อเปิดเสร็จแล้ว ให้คลิกที่หัวข้อ "Configuration" (การกำหนดค่า) ที่แถบเมนูด้านซ้าย
  5. กรอกรายละเอียดข้อมูลบอตดังนี้:
    • App name: ชื่อบอตของคุณ เช่น TaskyBot
    • Avatar URL: ลิงก์รูปภาพสัญลักษณ์ของบอต
    • Description: รายละเอียดบอต เช่น "บอตช่วยจัดการบอร์ดงาน Jira และแจ้งเตือนทีม"
  6. เลื่อนลงมาตรงหัวข้อ **Interactive Connection Settings** ให้คลิกเปิด **"Connection settings"**
  7. คลิกเลือกที่หัวข้อ **"Webhook URL" (หรือ HTTPS Endpoint)**
  8. นำลิงก์แอปพลิเคชันหลังบ้านของคุณที่ Deploy บน **Vercel** มากอกลงไป โดยลงท้ายลิงก์ด้วยคำว่า /api/webhook เสมอ
    ตัวอย่างการกรอก: https://your-app.vercel.app/api/webhook
  9. ส่วนล่างสุดหัวข้อ **Visibility** ให้เลือกติ๊ก "Only people and groups in my domain..." เพื่อจำกัดการเข้าใช้งานเฉพาะในบริษัท
  10. กด **"Save" (บันทึก)** ข้อมูลทั้งหมดเป็นอันเสร็จเรียบร้อย

4. วิธีดึงค่า Supabase Database URL (ฐานข้อมูลระบบ)

ระบบนี้ต้องการฐานข้อมูลสำหรับเก็บบันทึกประวัติ รายชื่อเล่นทีม และสถานะของตั๋วงานเพื่อนำไปแสดงผลบน Dashboard

  1. เข้าสู่ระบบที่ Supabase.com
  2. คลิกเปิดโปรเจกต์ของคุณขึ้นมา (หากยังไม่มี ให้กดสร้าง New Project)
  3. คลิกที่ไอคอน **Settings (รูปฟันเฟือง)** ที่แถบด้านซ้ายล่างสุด -> เลือกหัวข้อ **"Database"**
  4. เลื่อนหน้าจอลงมาหาหัวข้อ **Connection string** จากนั้นคลิกแท็บ **"URI"**
  5. คัดลอกตัวหนังสือที่ขึ้นต้นด้วย postgres://...
  6. ให้แก้ไขเปลี่ยนข้อความตรงตัวหนังสือ [YOUR-PASSWORD] ให้เป็นรหัสผ่านฐานข้อมูลจริงที่คุณกำหนดไว้ตอนสร้างโปรเจกต์

5. สรุปตัวแปรทั้งหมดที่ต้องกรอก (.env)

เมื่อรวบรวมค่าคีย์ต่างๆ จาก 4 ขั้นตอนด้านบนมาได้แล้ว ให้นำมากรอกใส่ในไฟล์ .env หรือใส่ในหน้า Environment Variables บนหน้า Vercel เพื่อใช้รันระบบจริง ดังนี้ครับ:

# 1. ลิงก์ฐานข้อมูล Supabase DATABASE_URL="postgres://postgres.abc:Password123@aws-region.pooler.supabase.com:6543/postgres" # 2. ข้อมูลการเข้าใช้งาน Jira JIRA_DOMAIN="mycompany.atlassian.net" JIRA_EMAIL="your-email@company.com" JIRA_API_TOKEN="ATATT..." JIRA_PROJECT_KEY="TES" # 3. ลิงก์แจ้งเตือนเข้าห้อง Google Chat GOOGLE_CHAT_WEBHOOK_URL="https://chat.googleapis.com/v1/spaces/..." # 4. ตั้งค่าบริการปัญญาประดิษฐ์ Google Gemini LLM_PROVIDER="gemini" GEMINI_API_KEY="AQ.xxxxxxx" GEMINI_MODEL="gemini-3.5-flash" BOT_NAME="TaskyBot" # 5. รหัสลับสำหรับระบบความปลอดภัย (สุ่มตัวอักษรอะไรก็ได้ยาวๆ) SESSION_SECRET="random_session_key_here_12345" CRON_SECRET="random_cron_key_here_12345"
