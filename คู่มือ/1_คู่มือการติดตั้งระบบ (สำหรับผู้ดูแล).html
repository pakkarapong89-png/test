📋 คู่มือตั้งค่าระบบด้วยตนเอง (Manual Setup)

คำแนะนำทีละขั้นตอนสำหรับตั้งค่าระบบ ปกติ ด้วยตัวแปรและการรันคำสั่งโดยตรง
⚙️ คู่มือสำหรับการติดตั้งแบบปกติ: หน้านี้จัดทำขึ้นเพื่อให้คุณตั้งค่าระบบบอตและแดชบอร์ดด้วยขั้นตอนมาตรฐานสำหรับนักพัฒนาทั่วไป โดยไม่มีการใช้สคริปต์อัตโนมัติ CMD ใดๆ เพื่อความยืดหยุ่นในการปรับแต่งส่วนหลังบ้านทั้งหมดครับ
🧭 สารบัญขั้นตอนการติดตั้ง:
  • 🔑 สเต็ปที่ 1: การขอข้อมูลและคีย์ API สำคัญ
  • ⚡ ทางเลือก: การตั้งค่าอัตโนมัติผ่าน CMD (แนะนำ - รวดเร็ว)
  • 🗄️ สเต็ปที่ 2: การสร้างตารางฐานข้อมูลใน Supabase
  • 📝 สเต็ปที่ 3: การเขียนไฟล์ค่ากำหนดสภาพแวดล้อม (.env)
  • 🚀 สเต็ปที่ 4: การ Deploy ขึ้นระบบจริงบน Vercel
  • 🔌 สเต็ปที่ 5: การเชื่อมระบบ Webhooks
  • 🏁 สเต็ปที่ 6: การเริ่มดึงข้อมูลครั้งแรก (First Sync)
🔑 สเต็ปที่ 1: การขอข้อมูลและคีย์ API สำคัญ (ทีละขั้นตอนอย่างละเอียด)

ให้เปิดล็อกอิน (Login) บัญชีผู้ใช้งานค้างไว้ในเว็บเบราว์เซอร์ และคัดลอกค่ารหัสต่างๆ ด้านล่างนี้ไปจดลงในโปรแกรม Notepad บนเครื่องคอมพิวเตอร์ของคุณเตรียมไว้ครับ:

📌 คำแนะนำสำคัญเรื่องประเภทบัญชีที่ใช้เชื่อมต่อ (Recommended Platform Accounts):
เพื่อป้องกันไม่ให้ระบบหยุดชะงักในภายหลังและเพื่อให้เป็นไปตามมาตรฐานความปลอดภัยไอทีขององค์กร ควรจัดเตรียมประเภทบัญชีในการเชื่อมต่อแต่ละระบบดังนี้:
  • 1. บัญชี Jira (Atlassian): ต้องใช้บัญชีระดับ Jira Administrator หรือบัญชีตัวแทนระบบบอต (Service Account) ที่มีสิทธิ์เข้าถึง (Read/Write) โปรเจกต์ที่ต้องการดึงตั๋วงาน เพื่อความสามารถในการสร้าง API Token และเปิดใช้งาน Webhook ได้
  • 2. บัญชี GitHub: ควรใช้บัญชีทีมไอทีส่วนกลางของบริษัท (Corporate Shared Account) หรือบัญชี Developer หลักขององค์กร เพื่อเป็นที่จัดเก็บซอร์สโค้ดในรูปแบบ Private Repository และใช้เชื่อมต่อระบบ Build อัตโนมัติกับ Vercel
  • 3. บัญชี Google Cloud Console: ต้องใช้บัญชีอีเมลบริษัทที่เป็นสมาชิกของ **Google Workspace** หรือมีสิทธิ์เป็น **Google Workspace Admin** ขององค์กร (ห้ามใช้บัญชีส่วนตัว @gmail.com เด็ดขาด) เพื่อความปลอดภัยและการจำกัดสิทธิ์ให้ใช้งานได้เฉพาะคนในองค์กร
  • 4. บัญชี Google Chat: บัญชีผู้ใช้ที่เข้าไปกดสร้าง Webhook URL ในห้องแชท (Space) จะต้องเป็นผู้ที่มีบทบาทเป็น **Space Manager (ผู้จัดการพื้นที่ทำงาน)** หรือสมาชิกที่ได้รับอนุญาตให้จัดการแอปพลิเคชันภายนอก (Apps & Integrations) ในห้องแชทนั้นๆ
  • 5. บัญชี Vercel / Supabase (Hosting & DB): แนะนำให้สมัครและผูกข้อมูลผ่านบัญชีผู้ดูแลระบบไอที (IT Admin) หรือบัญชีอีเมลส่วนกลางของบริษัทเพื่อไม่ให้มีปัญหาเมื่อมีพนักงานลาออก
1.1 ลิงก์เชื่อมฐานข้อมูล Supabase (DATABASE_URL)

จุดประสงค์: ใช้เพื่อให้บอตสามารถเชื่อมต่อและเก็บบันทึกประวัติตั๋วงานทั้งหมดได้

  1. เข้าเว็บไซต์ Supabase.com -> คลิกปุ่มสีเขียว "Start your project" หรือสมัครสมาชิกผ่านบัญชี GitHub ให้เรียบร้อย
  2. เมื่อเข้ามาหน้าแรก ให้คลิกปุ่มสีเขียว "New Project" (สร้างโครงการใหม่) -> เลือกชื่อองค์กรของคุณ
  3. กรอกรายละเอียดโครงการใหม่:
    • Name (ชื่อโปรเจกต์): พิมพ์ตั้งชื่อตามชอบ เช่น Tasky-Jira-Bot
    • Database Password: ⚠️ สำคัญมาก: ตั้งรหัสผ่านฐานข้อมูลที่มีความปลอดภัย แล้วก๊อปปี้รหัสผ่านจริงตัวนี้จดแยกใส่ Notepad เก็บไว้ทันที (เพราะระบบจะไม่แสดงรหัสนี้ให้เห็นอีก และต้องนำไปใช้แทนที่ในภายหลัง)
    • Region: เลือกเป็น Singapore (ap-southeast-1) (เพื่อให้เซิร์ฟเวอร์อยู่ใกล้ไทยและทำงานได้ไวที่สุด)
    • Pricing Plan: เลือกเป็น Free
  4. คลิกปุ่มสีเขียว "Create new project" -> รอระบบเตรียมการประมาณ 1-2 นาที (สังเกตไอคอนวงกลมหมุนๆ จนกระทั่งสร้างโครงการเสร็จสมบูรณ์)
  5. เมื่อเข้ามาที่หน้าแรกของโครงการแล้ว มองไปที่แถบสีดำด้านบนสุดขวามือ คลิกปุ่มสีเขียวคำว่า "Connect" (อยู่ข้างชื่อโครงการของคุณ)
  6. ในป๊อปอัปสี่เหลี่ยมที่เด้งขึ้นมา ให้คลิกเลือกแท็บคำว่า "Direct" ที่อยู่ด้านบนสุด
  7. ในหน้าจอของแท็บ Direct ให้ระบุตั้งค่า 2 ข้อดังนี้:
    • Connection Method: ติ๊กเลือกที่หัวข้อ "Transaction pooler"
    • Type: คลิกเลือกที่คำว่า "URI"
  8. สังเกตกล่องสีดำที่มีลิงก์ขึ้นต้นด้วย postgres://... ให้คลิกปุ่ม "Copy" ที่อยู่มุมขวาของกล่องนั้นเพื่อคัดลอกลิงก์
  9. เอาลิงก์มาวางใน Notepad แล้วสังเกตตัวหนังสือคำว่า [YOUR-PASSWORD] ให้ลบคำนี้ออก (รวมทั้งวงเล็บเหลี่ยมด้วย) แล้วนำ รหัสผ่านจริง ที่คุณตั้งไว้ในขั้นตอนแรกสุดมาพิมพ์ใส่แทนที่ตรงจุดนั้นเป๊ะๆ
1.2 ข้อมูลการล็อกอินและเชื่อมต่อกับ Jira Cloud
  1. JIRA_DOMAIN (โดเมนของบริษัท): เปิดหน้าเบราว์เซอร์ใช้งาน Jira บอร์ดหลักปกติ สังเกตแถบที่อยู่เว็บด้านบน (URL) ก๊อปปี้เอาเฉพาะท่อนโดเมน เช่น mycompany.atlassian.net (ห้ามพิมพ์ https:// หรือเครื่องหมายขีดทับ / ต่อท้ายเด็ดขาด)
  2. JIRA_EMAIL: กรอกที่อยู่อีเมลที่คุณใช้เข้าใช้งานระบบ Jira ของที่ทำงานปกติ
  3. JIRA_API_TOKEN (รหัสผ่านเชื่อมโยงข้อมูล):
    • เข้าหน้าจัดการความปลอดภัยของ Atlassian: Atlassian API Tokens
    • คลิกปุ่มสีน้ำเงิน "Create API token"
    • ตั้งชื่อป้ายกำกับว่า Jira-GoogleChat-Bot แล้วกด Create
    • หน้าต่างจะแสดงรหัสคีย์ที่ขึ้นต้นด้วย ATATT... ให้กดปุ่ม "Copy" เก็บเข้า Notepad ทันที (⚠️ คำเตือน: รหัสนี้จะแสดงครั้งเดียวตอนสร้างเสร็จเท่านั้น หากปิดไปแล้วต้องสร้างใหม่)
  4. JIRA_PROJECT_KEY (คีย์ย่อโครงการ): สังเกตชื่อรหัสตั๋วงานใน Jira ของคุณ เช่น ตั๋วงานมีชื่อรหัสว่า DEV-120 หรือ TES-99 ให้เอาเฉพาะส่วนหน้าขีดคือ DEV หรือ TES (พิมพ์ตัวพิมพ์ใหญ่ทั้งหมด)
1.3 ลิงก์ห้องแชท Google Chat Webhook URL

จุดประสงค์: เป็นท่อปลายทางเพื่อให้บอตสามารถส่งข้อความไปหาคนในทีมได้

  1. เปิดหน้าแอป Google Chat ในคอมพิวเตอร์ของคุณ
  2. เอาเมาส์ไปชี้ที่ชื่อห้องกลุ่ม (Space) ของทีมคุณ -> คลิกที่เครื่องหมายสามจุด ... (หรือคลิกที่ชื่อกลุ่มด้านบนสุดของหน้าจอแชท) -> เลือกเมนู "Apps & integrations" (แอปและการบูรณาการ)
  3. คลิกเลือกเมนูย่อย "Webhooks"
  4. คลิกปุ่ม "Add webhook" (หรือปุ่มเพิ่ม) -> ตั้งชื่อโปรไฟล์ของบอต เช่น Tasky Alert Bot -> คลิกปุ่ม Save (บันทึก)
  5. กดปุ่ม "Copy" ลิงก์ที่เด้งขึ้นมาเก็บไว้ (ขึ้นต้นด้วย https://chat.googleapis.com/v1/spaces/...)
1.4 รหัส Gemini API Key (คีย์สมองบอต AI)

จุดประสงค์: เพื่อใช้ปัญญาประดิษฐ์มาสรุปแจกแจงงานด่วนและวิเคราะห์ทีมงาน

  1. เปิดหน้าเว็บ Google AI Studio แล้วล็อกอินด้วยบัญชี Gmail/Google ทั่วไป
  2. มองหาแท็บเมนูด้านซ้ายบนสุด คลิกที่เมนูรูปไอคอนกุญแจเขียนว่า "Get API key"
  3. คลิกปุ่มสีน้ำเงินคำว่า "Create API key"
  4. หน้าต่างย่อยจะเด้งขึ้นมา ให้เลือกสร้างคีย์ในโปรเจกต์ใหม่ หรือเลือกโปรเจกต์เดิมที่มีอยู่ แล้วคลิกยืนยันสร้าง
  5. คลิกปุ่ม "Copy" รหัสคีย์ยาวๆ ที่ระบบสรุปให้มา (รหัสจะขึ้นต้นด้วยตัวอักษรย่อ AQ. หรือ AL. เสมอ) ก๊อปปี้ไปวางรอไว้ใน Notepad ครับ
⚡ ทางเลือกในการติดตั้ง: ใช้ตัวช่วยติดตั้งอัตโนมัติผ่าน CMD (ข้ามสเต็ปที่ 2 และ 3 ได้ทันที)

หากคุณไม่ต้องการทำตามขั้นตอนสร้างตารางฐานข้อมูลและเขียนไฟล์ตั้งค่าด้วยตนเอง ผู้พัฒนาได้เตรียมสคริปต์หน้าต่างคำสั่ง (CMD Tool) เพื่อช่วยอำนวยความสะดวกในการตั้งค่าแบบตอบคำถามสั้นๆ ไว้ให้แล้วครับ:

💡 ประโยชน์ของการติดตั้งผ่าน CMD: ระบบจะทำการตรวจสอบโปรแกรม Node.js ในเครื่องคอมพิวเตอร์ของคุณ ดาวน์โหลดไลบรารีที่เกี่ยวข้องทั้งหมด เชื่อมต่อไปยังฐานข้อมูลเพื่อสร้างตารางให้โดยอัตโนมัติ สุ่มคีย์ความปลอดภัยอย่างแน่นหนา และเขียนลงไฟล์ .env ให้เองในคลิกเดียว!

🛠️ ขั้นตอนการรันโปรแกรมช่วยตั้งค่าผ่าน CMD:

  1. เปิดโฟลเดอร์โครงการ: เปิดโฟลเดอร์ส่งมอบงานที่คุณแตกไฟล์ไว้
  2. ดับเบิลคลิกไฟล์ติดตั้ง: ดับเบิลคลิกเปิดไฟล์ setup.bat (สำหรับ Windows) หรือรัน ./setup.sh บน macOS / Linux
  3. ตอบคำถามข้ามการเปิดหน้าเว็บ (ถ้ามีคีย์แล้ว):
    • สคริปต์จะถามว่า "คุณต้องการให้ระบบช่วยเปิดหน้าเว็บขอกุญแจ/คีย์อัตโนมัติหรือไม่? (Y/N)"
    • หากคุณก๊อปปี้คีย์เตรียมไว้เรียบร้อยแล้ว ให้กดพิมพ์ N แล้ว Enter ได้เลยครับ เพื่อไม่ให้หน้าเว็บเด้งรบกวน
    • หากยังไม่มีคีย์ ให้พิมพ์ Y แล้ว Enter ระบบจะช่วยเด้งเบราว์เซอร์เปิดหน้าเว็บขอรับคีย์ตามหัวข้อต่างๆ ให้เอง
  4. พิมพ์กรอกข้อมูลลับตามที่สคริปต์ถามทีละข้อ:
    • Supabase URI (DATABASE_URL): วางลิงก์เชื่อมต่อฐานข้อมูลจากหน้าเว็บ Supabase
    • ข้อมูล Admin: กำหนดชื่อผู้ใช้ (Admin Username) และรหัสผ่าน (Admin Password) สำหรับล็อกอินเข้า Dashboard ของคุณ
    • ข้อมูล Jira: พิมพ์กรอกโดเมน Jira, อีเมลของแอดมิน Jira, API Token, และคีย์ของโปรเจกต์ Jira
    • ข้อมูลแจ้งเตือน: วางลิงก์ Google Chat Webhook และกุญแจ Gemini API Key พร้อมกำหนดรุ่นโมเดลหลัก (กด Enter ผ่านเพื่อใช้ค่าเริ่มต้นแนะนำ)
  5. ตรวจสอบผลลัพธ์: เมื่อกรอกเสร็จ สคริปต์จะสร้างตารางฐานข้อมูลทั้งหมด 7 ตารางลงในฐานข้อมูล Supabase และสร้างไฟล์ตั้งค่า nextjs-app/.env ให้เองเสร็จสรรพ
  6. คัดลอกตัวแปรสรุป: หน้าจอ CMD จะแสดงตารางสรุปตัวแปรทั้งหมดทางตอนท้ายสุด ให้คัดลอกหรือแคปหน้าจอเก็บไว้เพื่อนำไปใช้ในขั้นตอน Deploy ขึ้นเว็บคลาวด์จริง (สเต็ปที่ 4) ถัดไปครับ
🎉 การข้ามขั้นตอน: เมื่อรันตัวช่วยติดตั้งผ่าน CMD นี้เรียบร้อยแล้ว คุณสามารถข้าม "สเต็ปที่ 2 (สร้างตาราง)" และ "สเต็ปที่ 3 (สร้างไฟล์ .env)" ในคู่มือด้านล่างนี้ และข้ามไปเริ่มทำต่อที่ "สเต็ปที่ 4 (Deploy บน Vercel)" ได้ทันทีครับ!
🗄️ สเต็ปที่ 2: การสร้างตารางฐานข้อมูลใน Supabase (Database Setup)

เนื่องจากเป็นขั้นตอนปกติ ให้คุณคัดลอกสคริปต์ SQL ด้านล่างนี้ไปสั่งรันตารางในหน้าเว็บโดยตรงครับ:

  2. คลิกสร้าง "New Query" หน้าเอกสารเปล่าใหม่

    • หากใช้ Windows 10: คลิกที่แท็บ View ด้านบนสุดของหน้าต่าง -> ติ๊กถูกที่ช่อง "File name extensions" (นามสกุลไฟล์)

  3. ค้นหาไฟล์ตัวอย่างที่ชื่อ .env.example -> คลิกขวาเลือก Copy (คัดลอก) และกด Paste (วาง) ลงในโฟลเดอร์เดิม

🚀 สเต็ปที่ 4: การ Deploy ขึ้นระบบจริงบน Vercel

จุดประสงค์: นำซอร์สโค้ดขึ้นเซิร์ฟเวอร์คลาวด์ เพื่อให้ระบบบอตและแดชบอร์ดเข้าถึงได้ตลอด 24 ชม.

4.1 อัปโหลดโค้ดขึ้น GitHub
  1. ล็อกอินเข้าเว็บ GitHub.com ด้วยบัญชีของคุณ
  2. คลิกปุ่ม "+" ที่มุมขวาบนสุด -> เลือก "New repository"
  3. กรอกรายละเอียด:
    • Repository name: ตั้งชื่อตามชอบ เช่น tasky-jira-bot
    • Visibility: เลือก "Private" (เพื่อไม่ให้คนอื่นเห็นรหัสลับ)
  4. คลิกปุ่มสีเขียว "Create repository"
  5. ทำตามคำสั่ง Git ที่ GitHub แสดงให้ เพื่ออัปโหลดไฟล์โค้ดทั้งหมดขึ้นไป หรือหากยังไม่เคยใช้ Git สามารถคลิกปุ่ม "uploading an existing file" แล้วลากไฟล์โปรเจกต์มาวางในหน้าต่างได้เลย
4.2 เชื่อมต่อ Vercel กับ GitHub
  1. เปิดเว็บ Vercel.com -> คลิก "Sign Up" หรือ "Log In" ด้วยบัญชี GitHub (เลือก "Continue with GitHub")
  2. คลิกปุ่มสีดำ "Add New..." ที่มุมขวาบน -> เลือก "Project"
  3. ระบบจะแสดงรายชื่อ Repository ของคุณ มองหาชื่อ Repository ที่เพิ่งสร้าง แล้วคลิกปุ่ม "Import" ด้านขวามือ
4.3 กรอกค่า Environment Variables
  1. หลังจากกด Import แล้ว อย่าเพิ่งกด Deploy ให้เลื่อนหน้าจอลงมาก่อน
  2. มองหาหัวข้อ "Environment Variables" (ตัวแปรสภาพแวดล้อม)
  3. นำค่าคีย์จากไฟล์ .env มากรอกใส่ ทีละคู่ โดย:
    • ช่อง "Key": พิมพ์ชื่อตัวแปรเช่น DATABASE_URL
    • ช่อง "Value": วางค่าจริงที่จดไว้
    • คลิกปุ่ม "Add" เพื่อเพิ่มคู่นี้
  4. ทำซ้ำจนครบทุกตัวแปร (DATABASE_URL, JIRA_DOMAIN, JIRA_EMAIL, JIRA_API_TOKEN, JIRA_PROJECT_KEY, GOOGLE_CHAT_WEBHOOK_URL, LLM_PROVIDER, GEMINI_API_KEY, GEMINI_MODEL, SESSION_SECRET, CRON_SECRET, BOT_NAME)
4.4 กด Deploy
  1. เมื่อกรอก Environment Variables ครบทุกตัวแล้ว คลิกปุ่มสีดำ/น้ำเงินคำว่า "Deploy" ด้านล่างสุด
  2. รอระบบ Build ประมาณ 1-3 นาที (สังเกตหน้าจอจะแสดง Log ของการ Build ขึ้นมาเรื่อยๆ)
  3. เมื่อขึ้นข้อความ "Congratulations!" พร้อมหน้าจอตัวอย่างเว็บ แสดงว่า Deploy สำเร็จ!
  4. คลิก "Continue to Dashboard" -> มองหาหัวข้อ "Domains" แล้วก๊อปปี้ลิงก์เว็บไซต์ของคุณ (ตัวอย่าง: https://tasky-jira-bot.vercel.app)

⚠️ สำคัญ: ลิงก์เว็บนี้จะต้องนำไปใช้ในสเต็ป 5 และ 6 ต่อไป ห้ามลืมก๊อปปี้เก็บไว้ครับ

🔌 สเต็ปที่ 5: การเชื่อมระบบ Webhooks
5.1 ตั้งค่าเว็บฮุคฝั่ง Jira Cloud:

เปิดลิงก์ตั้งค่านี้ในเบราว์เซอร์: https://[โดเมน_Jira_ของคุณ]/plugins/servlet/webhooks

  1. คลิกปุ่ม "Create a Webhook" ขวาบน
  2. ตั้งค่ารายละเอียดดังนี้:
    • Name: Sync to Dashboard
    • URL: ลิงก์ Vercel ของคุณต่อท้ายด้วย /api/jira/webhook (เช่น https://my-jira-bot.vercel.app/api/jira/webhook)
    • JQL Filter: ระบุคำสั่งสูตรกรอง project = "รหัสย่อโปรเจกต์ของคุณ" (เช่น project = "DEV")
    • Events: เลื่อนลงหาหมวด Issue ติ๊กถูกที่ช่อง Created, Updated, และ Deleted
  3. คลิกปุ่มสีน้ำเงินด้านล่างสุด "Create"
5.2 ตั้งค่าเว็บฮุคฝั่ง Google Chat API:

จุดประสงค์: เมื่อมีคนพิมพ์ข้อความหาบอตในห้องแชท ระบบจะส่งข้อความนั้นมาที่เว็บ Vercel ของเราเพื่อประมวลผลตอบกลับ

  1. เปิดเว็บ Google Cloud Console แล้วล็อกอินด้วยบัญชี Google Workspace ของบริษัท (บัญชีอีเมลองค์กรเดียวกันกับที่ใช้เล่น Google Chat ในที่ทำงาน)
  2. สร้างหรือเลือกโปรเจกต์: คลิกชื่อโปรเจกต์ด้านบนสุด -> หากยังไม่มีให้คลิก "NEW PROJECT" -> ตั้งชื่อเช่น Tasky Chat Bot -> คลิก "Create"
  3. เปิดใช้งาน Google Chat API: คลิกช่องค้นหาด้านบน พิมพ์ Google Chat API กด Enter -> คลิกเลือกผลลัพธ์ที่ชื่อ "Google Chat API" -> คลิกปุ่มสีน้ำเงิน "Enable"
  4. ตั้งค่าโปรไฟล์บอต: คลิกเมนู "Configuration" ด้านซ้าย -> กรอก App name: เช่น TaskyBot, Description: เช่น Jira ticket tracking bot
  5. ตั้งค่าการเชื่อมต่อ: เลื่อนลงมาหาหัวข้อ "Connection settings" -> ติ๊กเลือก "App URL" (หรือ "HTTP endpoint URL") -> ในช่องกรอก URL ให้วางลิงก์เว็บ Vercel แล้วพิมพ์ /api/webhook ต่อท้าย
    ตัวอย่าง: https://tasky-jira-bot.vercel.app/api/webhook
  6. ตั้งค่าสิทธิ์: หาหัวข้อ "Visibility" -> ติ๊กเลือก "Make this Chat app available to specific people..." ตามความเหมาะสม
  7. คลิกปุ่ม "Save" ล่างสุด
  8. ทดสอบ: กลับไปที่ Google Chat -> เปิดห้อง Space ของทีม -> พิมพ์ @TaskyBot สรุปงาน หากบอตตอบกลับมาได้ แสดงว่าตั้งค่าถูกต้อง!
