📁 ขั้นตอนที่ 0: รับไฟล์โค้ดและอัปโหลดขึ้น GitHub

❗ IMPORTANT
ขั้นตอนนี้ทำเพียงครั้งเดียว Vercel จะ Deploy อัตโนมัติทุกครั้งที่มีการแก้ไขโค้ดในอนาคต

0.1 สิ่งที่ได้รับมา

คุณจะได้รับโฟลเดอร์ชื่อ taskyapp-handoff ซึ่งมีโครงสร้างดังนี้:

taskyapp-handoff/
├── 📄 complete_setup_guide.pdf   ← คู่มือนี้
├── 🪟 setup.bat                  ← สคริปต์ตั้งค่า (Windows)
├── 🐧 setup.sh                   ← สคริปต์ตั้งค่า (Mac/Linux)
└── 📁 nextjs-app/                ← โค้ดแอปพลิเคชันทั้งหมด
    ├── 📄 README.md
    ├── 📄 .env.example            ← ตัวอย่างตัวแปรที่ต้องกรอก
    └── ... (source code)

0.2 สร้าง GitHub Repository ใหม่

1.เข้า 👉 https://github.com แล้วล็อกอิน (หรือสมัครฟรีถ้ายังไม่มีบัญชี)
2.คลิกปุ่ม + มุมขวาบน → เลือก New repository
3.ตั้งชื่อ Repository เช่น taskyapp
4.เลือก Private (แนะนำ) → กดปุ่ม Create repository

0.3 อัปโหลดโค้ดขึ้น GitHub (วิธีง่าย — ผ่านเว็บ ไม่ต้องใช้ Terminal)

1.หลังสร้าง Repo เสร็จ คลิกลิงก์ uploading an existing file ที่หน้า Repo
2.เปิดโฟลเดอร์ taskyapp-handoff/nextjs-app ในเครื่องของคุณ
3.เลือกไฟล์และโฟลเดอร์ ทั้งหมด ในนั้น แล้ว ลากวางลงในหน้า GitHub
4.เลื่อนลงล่าง กด Commit changes (สีเขียว)
5.รอสักครู่จนอัปโหลดเสร็จ
💡 ทางเลือก (ถ้าถนัด Terminal): เปิด Terminal ในโฟลเดอร์ taskyapp-handoff แล้วพิมพ์:
git remote set-url origin https://github.com/[ชื่อบัญชี]/[ชื่อ repo].git
git push -u origin main

🔑 ขั้นตอนที่ 1: เตรียม API Keys (เก็บไว้ใน Notepad ก่อน)

💡 TIP
เปิด Notepad หรือ Notepad++ เตรียมไว้ แล้วคัดลอกค่าที่ได้จากแต่ละบริการลงไป จะได้ไม่หาย

1.1 DATABASE_URL — ลิงก์เชื่อมต่อฐานข้อมูล Supabase

1.เข้า 👉 https://supabase.com → สมัคร/ล็อกอิน (ฟรี)
2.กด New project → ตั้งชื่อโครงการ → ตั้งรหัสผ่านฐานข้อมูล → เลือก Region: Southeast Asia (Singapore) → กด Create new project
3.รอสัก 1 นาทีให้โครงการสร้างเสร็จ
4.กดปุ่ม Connect (สีเขียว บริเวณด้านบนสุด)
5.เลือก Transaction pooler → ประเภท URI
6.คัดลอกลิงก์ที่ขึ้นต้นด้วย postgresql://...
7.แก้ [YOUR-PASSWORD] ในลิงก์เป็นรหัสผ่านที่ตั้งไว้ → บันทึกลง Notepad

1.2 ข้อมูล Jira Cloud (4 ค่า)

1.ล็อกอิน Jira Cloud ขององค์กร
2.สร้าง API Token ที่ 👉 https://id.atlassian.com/manage-profile/security/api-tokens
3.กด Create API token → ตั้งชื่อว่า taskyapp-connector → คัดลอก Token บันทึกลง Notepad

บันทึกค่าทั้ง 4 ลง Notepad:

JIRA_DOMAIN      = your-company.atlassian.net
JIRA_EMAIL       = อีเมลของคุณที่ใช้ใน Jira
JIRA_API_TOKEN   = (Token ที่เพิ่งคัดลอกมา)
JIRA_PROJECT_KEY = รหัสย่อโครงการ เช่น KAN หรือ DEV
💡 หา Project Key ได้จาก Jira → เลือกโครงการ → ดูตัวย่อที่ปรากฏหน้า Issue เช่น KAN-1 → Key คือ KAN

1.3 GOOGLE_CHAT_WEBHOOK_URL — ลิงก์ส่งการแจ้งเตือน

1.เปิด Google Chat → คลิกเข้าห้องกลุ่ม (Space) ที่ต้องการรับการแจ้งเตือน
2.คลิกที่ชื่อห้องด้านบนสุด → เลือก Apps & integrations
3.เลือก Webhooks → Add webhook
4.ตั้งชื่อว่า taskyapp Notifier → กด Save
5.คัดลอก URL ที่ได้ → บันทึกลง Notepad

1.4 GEMINI_API_KEY — รหัส AI

1.เข้า 👉 https://aistudio.google.com/apikey
2.กด Create API Key
3.คัดลอก Key (ขึ้นต้นด้วย AIzaSy...) → บันทึกลง Notepad

⚡ ขั้นตอนที่ 2: รันสคริปต์ตั้งค่าฐานข้อมูล (อัตโนมัติ)

❗ IMPORTANT
ขั้นตอนนี้จะสร้างตารางฐานข้อมูลและรหัสผ่านสำหรับ Login Dashboard ให้อัตโนมัติ

สำหรับ Windows:

1.เปิดโฟลเดอร์ taskyapp-handoff (โฟลเดอร์นอกสุด)
2.ดับเบิ้ลคลิกไฟล์ setup.bat
3.กรอกข้อมูลตามที่สคริปต์ถาม (ใช้ข้อมูลที่เก็บไว้ใน Notepad)

สำหรับ Mac/Linux:

1.เปิด Terminal ในโฟลเดอร์ taskyapp-handoff
2.พิมพ์: ./setup.sh แล้วกด Enter
3.กรอกข้อมูลตามที่สคริปต์ถาม

สคริปต์จะสร้างให้อัตโนมัติ:

•✅ ตารางฐานข้อมูลทั้งหมด 7 ตาราง
•✅ รหัส SESSION_SECRET และ CRON_SECRET (เก็บค่านี้ไว้ด้วย!)
•✅ บัญชีผู้ดูแลระบบ (Admin) สำหรับ Login Dashboard
•✅ ไฟล์ .env พร้อมค่าทั้งหมดสำหรับนำไปใส่ใน Vercel

🚀 ขั้นตอนที่ 3: Deploy แอปขึ้น Vercel

3.1 สร้างโปรเจกต์ใน Vercel

1.เข้า 👉 https://vercel.com → ล็อกอินด้วย GitHub Account
2.กด Add New → Project
3.เลือก Repository taskyapp ที่สร้างไว้ → กด Import

3.2 ตั้งค่า Root Directory ⚠️ สำคัญมาก

1.ในหน้า Configure Project ให้มองหาช่อง Root Directory
2.กด Edit → พิมพ์ว่า nextjs-app → กด ✓

3.3 กรอก Environment Variables

1.เลื่อนลงมาที่ส่วน Environment Variables
2.เปิดไฟล์ .env ที่ได้จาก setup.bat ด้วย Notepad
3.กด Ctrl+A เพื่อเลือกทั้งหมด → Ctrl+C คัดลอก
4.คลิกช่องแรก Key ในหน้า Vercel → Ctrl+V วาง
5.Vercel จะกรอก Environment Variables ทั้งหมดให้อัตโนมัติใน 1 วินาที ✅
❗ IMPORTANT
ถ้ายังไม่มีไฟล์ .env จากสคริปต์ ให้กรอกทีละบรรทัดตามตัวอย่างในไฟล์ .env.example

3.4 Deploy

1.กดปุ่ม Deploy (สีดำ)
2.รอประมาณ 1–2 นาที จนขึ้นว่า Congratulations!
3.จดลิงก์เว็บที่ได้ไว้ เช่น https://taskyapp-xxx.vercel.app

🏥 ขั้นตอนที่ 4: ตรวจสอบระบบและซิงค์ข้อมูลครั้งแรก

4.1 ตรวจสอบว่าระบบพร้อมทำงาน

เปิดเบราว์เซอร์ไปที่:

https://[ลิงก์เว็บของคุณ]/api/health

ระบบจะแสดงผลรายงานเป็นภาษาไทย:

•✅ "overall": "ready" → ระบบพร้อมใช้งานทุกส่วน
•⚠️ "overall": "needs_attention" → มีบางส่วนที่ต้องตรวจสอบ (อ่านรายละเอียดในผลลัพธ์)

4.2 ดึงข้อมูลจาก Jira เข้าระบบครั้งแรก

เปิดเบราว์เซอร์ไปที่:

https://[ลิงก์เว็บของคุณ]/api/cron/sync-tickets?secret=[CRON_SECRET]
💡 หาค่า CRON_SECRET ได้จากไฟล์ .env หรือใน Vercel → Settings → Environment Variables

ผลลัพธ์ที่ถูกต้อง:

{"success": true, "message": "Synchronized 10 tickets successfully.", "count": 10}

หลังจากนั้น รีเฟรชหน้า Dashboard ข้อมูลตั๋วงานทั้งหมดจาก Jira จะแสดงขึ้นมาครับ

🤖 ขั้นตอนที่ 5: ตั้งค่า Google Chat Bot

5.1 เปิดใช้งาน Google Chat API

1.เข้า 👉 https://console.cloud.google.com
2.ล็อกอินด้วยอีเมลองค์กร → เลือกโครงการของคุณ
3.ค้นหา Google Chat API ในช่องค้นหาด้านบน → กด Enable

5.2 กำหนดค่าบอท

1.คลิกเมนูซ้าย Configuration แล้วกรอกข้อมูล:
ช่องค่าที่กรอก
App nametaskyapp (ชื่อนี้จะแสดงในแชท — เปลี่ยนเป็นชื่ออื่นได้)
Interactive featuresเปิดเป็นสีฟ้า ✅
Receive 1:1 messagesติ๊กถูก ✅
Join spaces and group conversationsติ๊กถูก ✅
Connection settingsเลือก HTTP Endpoint URL
URLhttps://[ลิงก์เว็บ Vercel]/api/webhook
VisibilityEveryone in your organization
2.กดปุ่ม Save ด้านล่างสุด

5.3 เพิ่มบอทเข้าใช้งาน

วิธีที่ 1 — คุยส่วนตัว (DM):

1.เปิด Google Chat → กด + ด้านซ้าย → Find apps
2.พิมพ์ taskyapp → เลือกบอท → เริ่มพิมพ์คำสั่งได้เลย (ไม่ต้อง @)

วิธีที่ 2 — ดึงเข้าห้องกลุ่ม (Space):

1.คลิกเข้าห้องแชทกลุ่ม → คลิกชื่อห้องด้านบน
2.เลือก Manage members → + Add people & apps
3.พิมพ์ taskyapp → กดเพิ่ม
4.บอทจะส่งข้อความต้อนรับพร้อมคู่มือสั่งงานให้ทันที
❗ IMPORTANT
ในห้องกลุ่ม ต้องพิมพ์ @taskyapp นำหน้าคำสั่งเสมอ เช่น @taskyapp ดูงานค้างทั้งหมด

⚙️ ขั้นตอนที่ 6: ตั้งค่า Jira Webhook (รับการแจ้งเตือนแบบ Real-time)

1.ล็อกอิน Jira ด้วยสิทธิ์ Admin → คลิกไอคอนเฟือง Settings ⚙️ → System
2.เมนูซ้าย เลื่อนลงล่างสุด → คลิก Webhooks
3.กด Create a Webhook มุมขวาบน
4.กรอกข้อมูล:
ช่องค่าที่กรอก
Nametaskyapp-jira-notifier
StatusEnabled
URLhttps://[ลิงก์เว็บ Vercel]/api/jira/webhook
JQLproject = [PROJECT_KEY] เช่น project = KAN
Events → Issueติ๊กทั้ง 3: created, updated, deleted
5.กด Create ด้านล่าง

🛠️ ขั้นตอนที่ 8: การแก้ไขปัญหาที่พบบ่อย

❌ ปัญหา: หน้าเว็บขึ้น Error 404

สาเหตุ: ลืมตั้ง Root Directory เป็น nextjs-app

วิธีแก้:

1.Vercel → เลือกโปรเจกต์ → Settings → General
2.หา Root Directory → กด Edit → พิมพ์ nextjs-app → Save
3.ไปที่ Deployments → คลิก ... → Redeploy

❌ ปัญหา: Dashboard แสดงข้อมูลเป็น 0 ทั้งหมด

สาเหตุ: ยังไม่ได้ซิงค์ข้อมูลจาก Jira ครั้งแรก

วิธีแก้: เปิดเบราว์เซอร์ไปที่:

https://[ลิงก์เว็บ]/api/cron/sync-tickets?secret=[CRON_SECRET]

แล้วรีเฟรชหน้า Dashboard

❌ ปัญหา: Google Cloud แจ้ง "Error saving configuration"

สาเหตุ: อีเมลในช่อง Visibility ไม่ถูกโดเมนองค์กร หรือไม่ได้กด Enter หลังพิมพ์อีเมล

วิธีแก้:

1.ใช้อีเมลที่ลงท้ายด้วยโดเมนองค์กรเท่านั้น (ห้ามใช้ @gmail.com)
2.หลังพิมพ์อีเมล กด Enter ก่อนแล้วค่อย Save

❌ ปัญหา: บอทไม่ตอบสนองในห้องกลุ่ม

สาเหตุ: ลืมพิมพ์ @taskyapp นำหน้าคำสั่ง

วิธีแก้: ในห้องกลุ่มต้องพิมพ์ @taskyapp นำหน้าเสมอ ถ้าคุยผ่าน DM ไม่ต้อง @

❌ ปัญหา: ระบบ /api/health แสดงว่า Jira เชื่อมต่อไม่ได้

วิธีแก้:

1.ตรวจสอบค่า JIRA_DOMAIN, JIRA_EMAIL, JIRA_API_TOKEN, JIRA_PROJECT_KEY ใน Vercel Environment Variables
2.ตรวจว่า API Token ยังไม่หมดอายุที่ https://id.atlassian.com/manage-profile/security/api-tokens

🔧 การเปลี่ยนชื่อบอทในอนาคต

ถ้าต้องการเปลี่ยนชื่อบอทให้ทำ 2 จุดนี้ให้ตรงกัน:

1.Google Cloud Console → Google Chat API → Configuration → App name = ชื่อใหม่
2.Vercel → Settings → Environment Variables → BOT_NAME = ชื่อใหม่ → Redeploy
