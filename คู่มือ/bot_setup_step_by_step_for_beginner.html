คู่มือแบบจับมือทำ: ติดตั้งและใช้งาน Bot Google Chat + Jira + Dashboard

ฉบับอ่านง่าย: ทำตามทีละขั้น พร้อมบอกว่าค่าแต่ละตัวเอาจากไหน เอาไปใส่ตรงไหน และตรวจอย่างไร
# คู่มือแบบจับมือทำ: ติดตั้งและใช้งาน Bot Google Chat + Jira + Dashboard

คู่มือนี้เขียนสำหรับคนที่ยังไม่แน่ใจว่าแต่ละขั้นต้องทำอะไร เปิดเว็บไหน กดตรงไหน และเอาค่าที่ได้ไปใส่ที่ใด ให้ทำตามจากบนลงล่าง ไม่ต้องข้ามขั้น

## อ่านตรงนี้ก่อนเริ่ม

ระบบนี้มี 3 ส่วนที่ต้องเชื่อมกัน

1. Jira: เป็นที่เก็บงานจริง เช่น ticket, assignee, status
2. Google Chat Bot: ให้คนในทีมพิมพ์สั่ง bot ใน Google Chat
3. Dashboard บน Vercel: เป็นเว็บและ API กลาง เชื่อม Google Chat, Jira, Supabase และ Gemini

ภาพรวมการไหลของระบบ

```text
ผู้ใช้พิมพ์ใน Google Chat
        ↓
Google Chat App ส่งข้อความมาที่ /api/webhook บน Vercel
        ↓
ระบบใช้ Gemini อ่านคำสั่ง แล้วเรียก Jira API
        ↓
Jira สร้าง/แก้/ปิด ticket
        ↓
Dashboard แสดงข้อมูล และ Google Chat รับแจ้งเตือนกลับ
```

สิ่งสำคัญมาก

- ห้ามส่งไฟล์ `.env`, `.env.local`, API key, token, database URL หรือ webhook URL ขึ้น GitHub
- ถ้าเผลอ push secret ขึ้น GitHub ให้เปลี่ยน token/key ทันที ไม่ใช่แค่ลบไฟล์ออก
- ค่าในเครื่อง local กับค่าใน Vercel ต้องเหมือนกันในส่วนที่ระบบต้องใช้จริง

## Checklist สั้น ๆ ว่าต้องทำอะไรบ้าง

ให้ทำตามลำดับนี้

1. สร้าง database ที่ Supabase แล้วเอา `DATABASE_URL`
2. เตรียม Jira แล้วเอา `JIRA_DOMAIN`, `JIRA_EMAIL`, `JIRA_API_TOKEN`, `JIRA_PROJECT_KEY`
3. สร้าง Gemini API key แล้วเอา `GEMINI_API_KEY`
4. สร้าง Google Chat Incoming Webhook แล้วเอา `GOOGLE_CHAT_WEBHOOK_URL`
5. ใส่ค่าทั้งหมดในไฟล์ `.env` บนเครื่อง
6. รันระบบบนเครื่อง แล้วเปิด `/api/health` ตรวจสถานะ
7. push code ขึ้น GitHub
8. deploy ผ่าน Vercel
9. ใส่ Environment Variables บน Vercel
10. เอา URL หลัง deploy ไปตั้งใน Google Chat App และ Jira Webhook
11. ทดสอบ bot, dashboard, webhook และ daily summary

## 1. เตรียม Supabase และหา DATABASE_URL

### เป้าหมายของขั้นนี้

เราต้องสร้าง database สำหรับเก็บข้อมูลของระบบ เช่น ticket cache, logs, team members และข้อมูลที่ dashboard ใช้แสดงผล

ค่าที่ต้องได้จากขั้นนี้คือ

```env
DATABASE_URL=postgres://...
```

### วิธีทำทีละขั้น

1. เปิดเว็บ Supabase แล้วเข้าสู่ระบบ
2. กดสร้าง Project ใหม่
3. ตั้งชื่อ project เช่น `jira-chatbot-dashboard`
4. เลือก region ที่ใกล้ผู้ใช้งาน เช่น Singapore ถ้าทีมอยู่ไทย
5. ตั้งรหัสผ่าน database แล้วเก็บไว้ให้ดี
6. รอจน project สร้างเสร็จ
7. เข้าไปที่หน้า Project Dashboard
8. กดปุ่ม Connect หรือเมนู Database connection
9. เลือก connection string แบบ Postgres
10. ถ้าจะ deploy บน Vercel แนะนำเลือก connection แบบ pooler / transaction mode
11. Copy connection string ออกมา
12. แทนที่ `[YOUR-PASSWORD]` ด้วยรหัสผ่าน database จริง

### ตัวอย่างค่าที่จะได้

```env
DATABASE_URL=postgresql://postgres.xxxxxx:your_password@aws-0-ap-southeast-1.pooler.supabase.com:6543/postgres
```

### ต้องจดอะไรไว้

| ค่า | เอามาจากไหน | เอาไปใส่ที่ไหน |
| --- | --- | --- |
| `DATABASE_URL` | Supabase connection string | `.env` และ Vercel Environment Variables |

### ตรวจว่าขั้นนี้ผ่านหรือยัง

ผ่านถ้า

- มีค่า `DATABASE_URL` แล้ว
- ค่าเริ่มด้วย `postgres://` หรือ `postgresql://`
- ในค่าไม่มีข้อความ `[YOUR-PASSWORD]` เหลืออยู่
- ถ้าใช้ Vercel ควรเป็น pooler transaction mode และมักใช้ port `6543`

### ปัญหาที่พบบ่อย

| อาการ | สาเหตุที่เป็นไปได้ | วิธีแก้ |
| --- | --- | --- |
| ต่อ database ไม่ได้ | ลืมแทน `[YOUR-PASSWORD]` | ใส่รหัสผ่านจริงใน connection string |
| deploy แล้วต่อ database ไม่ได้ | ใช้ direct connection แทน pooler | เปลี่ยนเป็น pooler transaction mode |
| password มีอักขระพิเศษ | URL encode ไม่ถูก | เปลี่ยน password ให้ง่ายขึ้น หรือ encode ตัวอักษรพิเศษ |

## 2. เตรียม Jira และหาค่า JIRA ต่าง ๆ

### เป้าหมายของขั้นนี้

เราต้องให้ bot สามารถอ่าน สร้าง แก้ และปิดงานใน Jira ได้

ค่าที่ต้องได้จากขั้นนี้คือ

```env
JIRA_DOMAIN=your-company.atlassian.net
JIRA_EMAIL=your-email@example.com
JIRA_API_TOKEN=your_api_token
JIRA_PROJECT_KEY=ABC
```

---

### 2.1 หา JIRA_DOMAIN

#### วิธีทำ

1. เปิด Jira บน browser
2. ดู URL ด้านบน เช่น

```text
https://mycompany.atlassian.net/jira/software/projects/ABC/boards/1
```

3. เอาเฉพาะ domain ตรงนี้

```env
JIRA_DOMAIN=mycompany.atlassian.net
```

ไม่ต้องใส่ `https://` และไม่ต้องใส่ path หลัง domain

#### ตรวจว่าถูกไหม

ถูก

```env
JIRA_DOMAIN=mycompany.atlassian.net
```

ผิด

```env
JIRA_DOMAIN=https://mycompany.atlassian.net
JIRA_DOMAIN=mycompany.atlassian.net/jira/software/projects/ABC
```

---

### 2.2 หา JIRA_PROJECT_KEY

#### วิธีทำแบบง่าย

1. เปิด project ใน Jira
2. ดูรหัส ticket ใน project เช่น `ABC-12`, `DEV-9`, `TES-101`
3. ตัวอักษรก่อนขีดคือ project key

ตัวอย่าง

```text
ABC-12  →  JIRA_PROJECT_KEY=ABC
DEV-9   →  JIRA_PROJECT_KEY=DEV
```

#### ตรวจว่าถูกไหม

ผ่านถ้า project key เป็นตัวอักษรสั้น ๆ เช่น `ABC`, `DEV`, `OPS` และตรงกับ ticket จริงใน Jira

---

### 2.3 หา JIRA_EMAIL

ให้ใช้อีเมล Atlassian account ของคนหรือ service account ที่จะให้ bot ใช้เรียก Jira

ตัวอย่าง

```env
JIRA_EMAIL=admin@company.com
```

ข้อแนะนำ

- ควรใช้ service account เช่น `jira-bot@company.com` ถ้ามี
- account นี้ต้องมีสิทธิ์ดู สร้าง และแก้ ticket ใน project ที่เลือก

---

### 2.4 สร้าง JIRA_API_TOKEN

#### วิธีทำ

1. เปิดหน้า Atlassian account security
2. ไปที่ API tokens
3. กด Create API token
4. ตั้งชื่อ token เช่น `google-chat-jira-bot`
5. กด Create
6. Copy token ทันที เพราะส่วนใหญ่จะแสดงให้ copy แค่ครั้งเดียว
7. เก็บไว้ใน password manager หรือไฟล์ note ชั่วคราวที่ปลอดภัย

นำไปใส่แบบนี้

```env
JIRA_API_TOKEN=ใส่_token_ที่_copy_มา
```

#### ตรวจว่าถูกไหม

ผ่านถ้า

- มี token แล้ว
- token ถูกสร้างจาก account เดียวกับ `JIRA_EMAIL`
- account นั้นมีสิทธิ์ใน Jira project

### ปัญหาที่พบบ่อยของ Jira

| อาการ | มักเกิดจาก | วิธีแก้ |
| --- | --- | --- |
| Jira ตอบ 401 | email/token ผิด | เช็กว่า `JIRA_EMAIL` เป็น account เดียวกับ token |
| Jira ตอบ 403 | account ไม่มีสิทธิ์ | เพิ่ม permission ใน Jira project |
| Jira ตอบ 404 | domain หรือ project key ผิด | ตรวจ `JIRA_DOMAIN` และ `JIRA_PROJECT_KEY` |

## 3. เตรียม Gemini API Key

### เป้าหมายของขั้นนี้

Gemini ใช้ช่วยอ่านข้อความจาก Google Chat เช่น ผู้ใช้พิมพ์ว่า “สร้างงานให้สมชายเรื่องแก้ bug login ภายในพรุ่งนี้” แล้วแปลงเป็นข้อมูลที่ระบบเอาไปสร้าง Jira ticket ได้

ค่าที่ต้องได้คือ

```env
GEMINI_API_KEY=your_gemini_api_key
```

### วิธีทำทีละขั้น

1. เปิด Google AI Studio
2. Login ด้วยบัญชี Google ที่จะใช้กับโปรเจกต์
3. ไปที่เมนู API keys
4. กด Create API key
5. เลือก Google Cloud project หรือสร้าง project ใหม่
6. Copy API key ที่ได้
7. นำไปใส่ใน `.env`

ตัวอย่าง

```env
GEMINI_API_KEY=AIzaSyxxxxxxxxxxxxxxxxxxxx
```

### ตรวจว่าขั้นนี้ผ่านหรือยัง

ผ่านถ้า

- มีค่า API key แล้ว
- ค่าไม่ใช่ placeholder เช่น `your_api_key_here`
- account มีสิทธิ์ใช้ Gemini API

### ถ้ากดสร้าง API key ไม่ได้

ให้เช็ก 3 อย่างนี้

1. บัญชี Google มีสิทธิ์ใน Google Cloud project หรือไม่
2. องค์กรปิดการสร้าง API key ไว้หรือไม่
3. ต้องเปิด billing หรือ API เพิ่มก่อนหรือไม่ ขึ้นกับ policy ขององค์กร

## 4. เตรียม Google Chat Incoming Webhook

### เป้าหมายของขั้นนี้

Incoming Webhook ใช้สำหรับให้ระบบส่งข้อความเข้า Google Chat เช่น แจ้งเตือนสร้าง ticket สำเร็จ หรือส่ง daily summary

ค่าที่ต้องได้คือ

```env
GOOGLE_CHAT_WEBHOOK_URL=https://chat.googleapis.com/v1/spaces/...
```

### วิธีทำทีละขั้น

1. เปิด Google Chat
2. เข้า Space ที่ต้องการให้ bot ส่งแจ้งเตือน
3. กดชื่อ Space หรือเมนูตั้งค่า
4. เลือก Apps & integrations หรือ Manage webhooks
5. กด Add webhook
6. ตั้งชื่อ webhook เช่น `Jira Dashboard Bot`
7. กด Save
8. Copy webhook URL
9. เอา URL ไปใส่ใน `.env`

### ตรวจว่าถูกไหม

ผ่านถ้า

- URL เริ่มด้วย `https://chat.googleapis.com/`
- webhook อยู่ใน Space ที่ต้องการรับแจ้งเตือนจริง
- คนในองค์กรอนุญาตให้ใช้ incoming webhook

### จุดที่คนมักสับสน

Google Chat มี 2 URL คนละหน้าที่

| สิ่งที่ตั้ง | ใช้ทำอะไร | เอาค่าไปใส่ตรงไหน |
| --- | --- | --- |
| Incoming Webhook URL | ให้ระบบส่งข้อความเข้า Space | `GOOGLE_CHAT_WEBHOOK_URL` |
| Chat App endpoint | ให้ Google Chat ส่งข้อความผู้ใช้มาหา bot | ตั้งใน Google Cloud Console หลัง deploy |

จำง่าย ๆ

- Webhook URL = ระบบส่งออกไปหา Chat
- App endpoint = Chat ส่งเข้ามาหาเรา

## 5. สร้างไฟล์ .env บนเครื่อง local

### เป้าหมายของขั้นนี้

เอาค่าทั้งหมดที่หาได้มาใส่ในไฟล์ `.env` เพื่อให้ระบบบนเครื่องเราใช้ได้

### วิธีทำ

ในโฟลเดอร์โปรเจกต์ ให้สร้างไฟล์ชื่อ `.env`

ตัวอย่างไฟล์ `.env`

```env
DATABASE_URL=postgresql://postgres.xxxxxx:password@aws-0-ap-southeast-1.pooler.supabase.com:6543/postgres
JIRA_DOMAIN=mycompany.atlassian.net
JIRA_EMAIL=jira-bot@company.com
JIRA_API_TOKEN=ใส่_token_จริง
JIRA_PROJECT_KEY=ABC
GEMINI_API_KEY=ใส่_gemini_key_จริง
GOOGLE_CHAT_WEBHOOK_URL=https://chat.googleapis.com/v1/spaces/xxxx/messages?key=xxxx&token=xxxx
NODE_ENV=development
```

### ตรวจว่า .env ถูกไหม

ให้เช็กทีละข้อ

- ชื่อไฟล์คือ `.env` ไม่ใช่ `.env.txt`
- แต่ละบรรทัดเป็นรูปแบบ `KEY=value`
- ไม่มีช่องว่างหน้า/หลังเครื่องหมาย `=`
- ค่า secret ไม่ได้อยู่ในเครื่องหมาย `< >`
- ไม่มี placeholder เช่น `your_token_here`
- ไฟล์ `.env` อยู่ใน `.gitignore`

### สำคัญมาก

ก่อน push code ให้ตรวจว่า `.env` ไม่ถูก stage

```bash
git status
```

ถ้าเห็น `.env` อยู่ในรายการที่จะ commit ให้หยุดก่อน และเอาออกจาก git ทันที

## 6. ทดสอบระบบบนเครื่อง local

### เป้าหมายของขั้นนี้

ทดสอบว่าค่าที่ใส่ใน `.env` ใช้งานได้ ก่อนเอาขึ้น Vercel

### วิธีทำ

1. เปิด terminal ที่โฟลเดอร์โปรเจกต์
2. ติดตั้ง dependencies ถ้ายังไม่เคยติดตั้ง

```bash
npm install
```

3. รันระบบ

```bash
npm run dev
```

4. เปิด browser ไปที่

```text
http://localhost:3000/api/health
```

### ดูผลลัพธ์ยังไง

ถ้าผ่าน ควรเห็นประมาณนี้

```json
{
  "status": "ok",
  "database": "ok",
  "jira": "ok"
}
```

ถ้าไม่ผ่าน ให้ดูข้อความว่า fail ที่ส่วนไหน

| fail ตรงไหน | ให้กลับไปดูขั้นไหน |
| --- | --- |
| database missing/error | ขั้น Supabase และ `DATABASE_URL` |
| jira error | ขั้น Jira domain/email/token/project key |
| gemini missing | ขั้น Gemini API key |
| google chat webhook missing | ขั้น Google Chat Incoming Webhook |

## 7. Push code ขึ้น GitHub

### เป้าหมายของขั้นนี้

เอา code ขึ้น GitHub เพื่อให้ Vercel ดึงไป deploy ได้

### ก่อน push ต้องเช็ก secret ก่อน

รันคำสั่งนี้

```bash
git status
```

ต้องไม่เห็นไฟล์เหล่านี้อยู่ในรายการ commit

```text
.env
.env.local
*.key
*.pem
```

### ขั้นตอน push ครั้งแรก

1. สร้าง repository ใหม่บน GitHub
2. กลับมาที่ terminal ในโปรเจกต์
3. ตรวจสถานะ

```bash
git status
```

4. เพิ่มไฟล์เข้า commit

```bash
git add .
```

5. commit

```bash
git commit -m "Initial bot dashboard setup"
```

6. ตั้ง remote repository

```bash
git remote add origin https://github.com/your-org/your-repo.git
```

7. push ขึ้น GitHub

```bash
git push -u origin main
```

ถ้า branch หลักชื่อ `master` ให้ใช้ `master` แทน `main`

### ถ้าเผลอ push .env ขึ้นไปแล้ว

ให้ทำทันที

1. ลบ `.env` ออกจาก repo
2. commit การลบไฟล์
3. เปลี่ยนค่า secret ทุกตัวที่หลุด เช่น Jira token, Gemini key, Supabase password, Google Chat webhook
4. ใส่ค่าใหม่ใน Vercel และ local

ห้ามคิดว่า “ลบ commit แล้วพอ” เพราะ secret อาจถูกดึงไปแล้ว

## 8. Deploy ขึ้น Vercel

### เป้าหมายของขั้นนี้

ให้ระบบทำงานบน URL จริง เช่น

```text
https://your-project.vercel.app
```

### วิธีทำทีละขั้น

1. เปิด Vercel
2. Login ด้วยบัญชีที่เชื่อม GitHub ได้
3. กด Add New Project
4. เลือก repository ที่ push ไป
5. ตรวจ Root Directory
6. ถ้า repository มีหลายโฟลเดอร์ และโปรเจกต์อยู่ใน `nextjs-app` ให้ตั้ง Root Directory เป็น

```text
nextjs-app
```

7. Framework ควรเป็น Next.js
8. ยังไม่ต้องกด deploy ถ้ายังไม่ได้ใส่ Environment Variables หรือกด deploy แล้วค่อยแก้ env แล้ว redeploy ก็ได้

## 9. ใส่ Environment Variables บน Vercel

### เป้าหมายของขั้นนี้

เอาค่าใน `.env` ไปใส่ใน Vercel เพราะ Vercel ไม่ได้อ่านไฟล์ `.env` ในเครื่องเรา

### วิธีทำทีละขั้น

1. เข้า Vercel project
2. ไปที่ Settings
3. ไปที่ Environment Variables

## 10. ตั้ง Google Chat App หลัง deploy

### เป้าหมายของขั้นนี้

ทำให้ผู้ใช้พิมพ์หา bot ใน Google Chat แล้วข้อความถูกส่งมาที่ระบบของเรา

ค่าที่ต้องใช้คือ URL หลัง deploy รวมกับ `/api/webhook`

ตัวอย่าง

```text
https://your-project.vercel.app/api/webhook
```

### วิธีทำโดยรวม

1. เปิด Google Cloud Console
2. เลือก project ที่ใช้ทำ Chat App
3. เปิด Google Chat API
4. ไปที่ Configuration
5. ตั้ง App name เช่น `Jira Assistant Bot`
6. ตั้ง Avatar ถ้าต้องการ
7. ในส่วน App URL หรือ HTTP endpoint ให้ใส่

```text
https://your-project.vercel.app/api/webhook
```

8. ตั้ง permission ว่าให้ใครใช้ bot ได้ เช่น เฉพาะในองค์กร
9. Save
10. เพิ่ม bot เข้า Google Chat Space
11. ลอง mention bot แล้วพิมพ์คำสั่งทดสอบ

### ตัวอย่างคำสั่งทดสอบ

```text
@Jira Assistant Bot ช่วยสร้างงาน ทดสอบระบบ bot ให้หน่อย priority medium
```

### ถ้า bot ไม่ตอบ

เช็กตามนี้

| จุดที่ต้องเช็ก | รายละเอียด |
| --- | --- |
| Endpoint URL | ต้องเป็น `/api/webhook` และเป็น URL ของ Vercel จริง |
| Deploy สำเร็จไหม | เปิด `/api/health` ดูก่อน |
| Permission | bot ถูกเพิ่มเข้า Space แล้วหรือยัง |
| Logs | ดู Vercel Functions Logs ว่ามี request เข้ามาไหม |

## 11. ตั้ง Jira Webhook

### เป้าหมายของขั้นนี้

ให้ Jira แจ้งกลับมาที่ dashboard เมื่อ ticket ถูกสร้าง แก้ไข หรือเปลี่ยนสถานะ เพื่อให้ข้อมูลใน dashboard sync มากขึ้น

URL ที่ต้องใช้คือ

```text
https://your-project.vercel.app/api/jira/webhook
```

### วิธีทำทีละขั้น

2. ไปที่ System
3. หาเมนู Webhooks
4. กด Create webhook
5. ตั้งชื่อ เช่น `Dashboard Sync Webhook`
6. ใส่ URL

```text
https://your-project.vercel.app/api/jira/webhook
```

7. เลือก event ที่ต้องการ เช่น issue created, issue updated, issue deleted
8. จำกัด project เป็น project key ที่ใช้จริง เช่น `ABC`
9. Save

### ตรวจว่าผ่านไหม

1. เปิด ticket ใน Jira
2. ลองแก้ summary หรือ status
3. เปิด Vercel logs
4. ควรเห็น request เข้า `/api/jira/webhook`
5. เปิด dashboard ดูว่าข้อมูลเปลี่ยนหรือไม่

## 12. Sync tickets ครั้งแรก

### เป้าหมายของขั้นนี้

ดึง ticket จาก Jira เข้ามาเก็บใน database เพื่อให้ dashboard มีข้อมูลเริ่มต้น

### วิธีทำ

เปิด URL นี้หลัง deploy

```text
https://your-project.vercel.app/api/cron/sync-tickets
```

หรือถ้าทดสอบบนเครื่อง local

```text
http://localhost:3000/api/cron/sync-tickets
```

### ตรวจว่าผ่านไหม

ผ่านถ้า

- endpoint ไม่ error
- dashboard เริ่มเห็น ticket
- database มีข้อมูล ticket cache/activity logs

ถ้าไม่ผ่าน ให้เช็ก Jira token, project key และ permission

## 15. ตารางรวมว่าแต่ละค่าเอาจากไหน

| ค่า | ไปเอาที่ไหน | วิธีดูแบบสั้น | เอาไปใส่ที่ไหน |
| --- | --- | --- | --- |
| `DATABASE_URL` | Supabase | Project → Connect → Postgres connection string | `.env`, Vercel env |
| `JIRA_DOMAIN` | Jira URL | เอาเฉพาะ `xxx.atlassian.net` | `.env`, Vercel env |
| `JIRA_EMAIL` | Atlassian account | อีเมลของ account ที่สร้าง token | `.env`, Vercel env |

| `JIRA_PROJECT_KEY` | Jira ticket | ตัวอักษรก่อนขีด เช่น `ABC` จาก `ABC-123` | `.env`, Vercel env |
| `GEMINI_API_KEY` | Google AI Studio | API keys → Create API key | `.env`, Vercel env |
| `GOOGLE_CHAT_WEBHOOK_URL` | Google Chat Space | Manage webhooks → Copy URL | `.env`, Vercel env |
| Chat App endpoint | Vercel | `https://...vercel.app/api/webhook` | Google Chat API config |
| Jira webhook endpoint | Vercel | `https://...vercel.app/api/jira/webhook` | Jira Webhooks |

## 16. ลำดับตรวจปัญหาแบบง่ายที่สุด

ถ้าระบบไม่ทำงาน ให้ไล่แบบนี้

1. เปิด `/api/health`
2. ถ้า health ฟ้อง env missing ให้กลับไปใส่ env ใน Vercel แล้ว redeploy
3. ถ้า database error ให้เช็ก `DATABASE_URL`
4. ถ้า Jira error ให้เช็ก `JIRA_DOMAIN`, `JIRA_EMAIL`, `JIRA_API_TOKEN`, `JIRA_PROJECT_KEY`
5. ถ้า bot ใน Chat ไม่ตอบ ให้เช็ก Google Chat App endpoint และ Vercel logs
6. ถ้าแจ้งเตือนไม่เข้า Space ให้เช็ก `GOOGLE_CHAT_WEBHOOK_URL`
7. ถ้า dashboard ไม่มี ticket ให้เรียก sync tickets ครั้งแรก

## 18. สรุปจำง่าย

ถ้าจำทั้งหมดไม่ได้ ให้จำสูตรนี้

```text
หา key ให้ครบ → ใส่ .env → ทดสอบ local → push GitHub → deploy Vercel → ใส่ env บน Vercel → เอา URL Vercel ไปตั้ง Google Chat/Jira → ทดสอบจริง
```

จุดที่คนพลาดบ่อยที่สุดมี 4 จุด

1. ใส่ `JIRA_DOMAIN` พร้อม `https://` ซึ่งผิด
2. ลืมใส่ Environment Variables บน Vercel หลัง deploy
3. สับสนระหว่าง Google Chat Incoming Webhook กับ Google Chat App endpoint
4. เผลอ push `.env` ขึ้น GitHub
