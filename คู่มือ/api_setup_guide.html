คู่มือการขอค่า API และการ Deploy ระบบบอต Jira + Google Chat

ฉบับสมบูรณ์ครอบคลุมทุกขั้นตอนการติดตั้ง — ทำตามได้ทันที ไม่สับสน
📌 สารบัญขั้นตอนการตั้งค่า
  • 1. การขอค่าคีย์และสิทธิ์เชื่อมต่อ API (API Credentials)
  • 2. การนำโค้ดขึ้นระบบคลาวด์และติดตั้งใช้งานจริง (Deploy to Vercel)
  • 3. การกำหนดค่าและตั้งค่า Webhooks (Jira & Google Chat App)
  • 4. การเริ่มระบบและซิงก์ข้อมูลดึงตั๋วงานครั้งแรก (First Sync)
  • 5. สรุปฟังก์ชันการทำงานของระบบและระบบเวลาเตือนงานค้าง (Cron)

1. วิธีขอคีย์และสิทธิ์เชื่อมต่อ API (API Credentials)

ขั้นตอนการขอและจัดเตรียมรหัสกุญแจสำคัญ 4 ดอกก่อนจะทำการรันระบบจริง:

1.1 Google Gemini API Key (สมองแปลคำพูดของบอต)

  1. เปิดเบราว์เซอร์ไปที่: Google AI Studio
  2. ล็อกอินด้วยบัญชี Google ของท่าน
  3. คลิกปุ่ม "Get API key" ที่แถบเครื่องมือซ้ายบน
  4. คลิกเลือกปุ่มน้ำเงิน "Create API key"
  5. คลิกสร้างในโปรเจกต์ใหม่ (**Create API key in new project**) หรือเลือกโปรเจกต์เดิมที่มีอยู่
  6. เมื่อระบบสร้างเสร็จแล้ว ให้กดปุ่ม "Copy" บันทึกรหัสคีย์เก็บไว้ทันที
    💡 อัปเดตคีย์รูปแบบใหม่: ปัจจุบันคีย์จะขึ้นต้นด้วยตัวย่อ "AQ." หรือ "AL." เสมอ สามารถนำรหัสตัวนี้ไปใส่ในโปรเจกต์เพื่อคุยกับโมเดลได้ทันทีครับ

1.2 ข้อมูลสิทธิ์การเชื่อมต่อ Jira Cloud

จัดเตรียมข้อมูล 4 ค่าสำคัญดังนี้:

  1. JIRA_DOMAIN (ที่อยู่โดเมน Jira): เปิดหน้าบอร์ด Jira และก๊อปปี้เอาเฉพาะโดเมนหลัก เช่น myteam.atlassian.net (ห้ามมี https:// หรือข้อความต่อท้าย)
  2. JIRA_EMAIL: อีเมลที่ท่านล็อกอินทำงานบอร์ดงาน Jira
  3. JIRA_API_TOKEN:
    • เปิดหน้าความปลอดภัยของ Atlassian: Atlassian API Token Manager
    • คลิกปุ่ม **"Create API token"** แล้วตั้งชื่อกำกับ (เช่น Jira-GoogleChat-Bot) จากนั้นกดปุ่ม Create
    • รหัสจะขึ้นต้นด้วย **"ATATT..."** ให้กด Copy บันทึกในที่ปลอดภัยทันที *(จะแสดงผลให้เห็นเพียงครั้งเดียวเท่านั้น)*
  4. JIRA_PROJECT_KEY: รหัสตัวย่อหน้าตั๋วงานโปรเจกต์ เช่น ตั๋วงานคือ TES-154 รหัสคือ TES (ตัวพิมพ์ใหญ่ทั้งหมด)

1.3 ขอลิงก์ส่งการแจ้งเตือน (Google Chat Webhook URL)

สำหรับให้ตัวเซิร์ฟเวอร์หลังบ้านบอต ยิงข้อความรายงานผลประจำวัน หรือเตือนเวลามีตั๋วงานค้างเข้าห้องแชทอัตโนมัติ

  1. เปิดแอปพลิเคชัน Google Chat หรือ Google Chat ใน Gmail
  2. คลิกเลือกเข้าไปในห้องแชท (Space) ที่ต้องการให้บอตส่งข้อความ
  3. คลิกที่ **ชื่อห้องแชทด้านบนสุด** -> เลือก **"Apps & integrations" (แอปและการบูรณาการ)** -> คลิกเลือก **"Webhooks" (เว็บฮุค)**
  4. คลิกปุ่ม **"Add webhook" (เพิ่มเว็บฮุค)**
  5. ตั้งชื่อโปรไฟล์เว็บฮุคบอต (เช่น Tasky Alert Bot) แล้วกดปุ่ม **"Save" (บันทึก)**
  6. คัดลอกค่าลิงก์ URL ที่ได้รับมาเก็บไว้ใช้งาน (ลิงก์จะขึ้นต้นด้วย https://chat.googleapis.com/...)

1.4 ขอลิงก์ฐานข้อมูล Supabase Database URL

  1. สมัครใช้งานและสร้างโปรเจกต์ใหม่บน **Supabase.com**
  2. มองหาแถบเมนูด้านบนสุดของโปรเจกต์ แล้วคลิกปุ่มสีเขียว "Connect"
  3. ในหน้าต่างที่เด้งขึ้นมา ให้เลือกแท็บ "Direct" ด้านบนสุด
  4. สังเกตหัวข้อ Connection Method ให้เลือกตัวเลือก "Transaction pooler"
  5. ที่ช่อง Type ด้านล่าง ให้เลือกตัวเลือกเป็น "URI"
  6. กดปุ่มคัดลอก (Copy) ลิงก์ที่ขึ้นต้นด้วย postgres://...
  7. แก้ไขลิงก์โดยนำรหัสผ่านฐานข้อมูลที่คุณตั้งไว้ไปแทนที่คำว่า [YOUR-PASSWORD] ด้วยรหัสผ่านจริงของคุณ

2. ขั้นตอนการนำโค้ดขึ้นระบบและรันใช้งานจริง (Deploy to Vercel)

เมื่อรวบรวมคีย์ API ครบถ้วนแล้ว ขั้นตอนถัดไปคือนำโค้ดขึ้นรันบนอินเทอร์เน็ตสาธารณะ:

2.1 อัปโหลดโค้ดขึ้น GitHub

  1. เข้าเว็บ **[GitHub.com](https://github.com/)** สร้าง Repository ของคุณเอง ตั้งชื่อ เช่น `jira-googlechat-bot`
  2. เลือกประเภทเป็น Private (ส่วนตัว) เสมอ เพื่อความปลอดภัย
  3. ทำการอัปโหลดโค้ดระบบทั้งหมดขึ้นไปเก็บไว้ใน Repository ดังกล่าว

2.2 การ Deploy ขึ้น Vercel

  1. ล็อกอินเข้าไปที่ **[Vercel.com](https://vercel.com/)** ผ่านสิทธิ์บัญชี GitHub ของท่าน
  2. คลิกปุ่ม **"Add New"** -> เลือก **"Project"**
  3. กดคลิกปุ่ม **"Import"** ตัวคลังโปรเจกต์ `jira-googlechat-bot` ของเราที่ดึงมาจาก GitHub
  4. เลื่อนหน้าจอลงมาหาหัวข้อ **Environment Variables** แล้วนำค่าคีย์ที่เราเตรียมไว้ทั้งหมดไปใส่ดังนี้:
ชื่อตัวแปร (Key) ค่าข้อมูลที่จะนำมาใส่ (Value)
DATABASE_URL ลิงก์เชื่อมฐานข้อมูล Supabase จากข้อ 1.4
JIRA_DOMAIN โดเมนโปรเจกต์ Jira เช่น myteam.atlassian.net จากข้อ 1.2
JIRA_EMAIL อีเมลทำงานของ Jira จากข้อ 1.2
JIRA_API_TOKEN รหัสผ่าน API Token จากข้อ 1.2
JIRA_PROJECT_KEY รหัสตัวย่อโปรเจกต์ เช่น TES จากข้อ 1.2
GOOGLE_CHAT_WEBHOOK_URL ลิงก์ส่งข้อความแจ้งเตือนห้องแชท จากข้อ 1.3
LLM_PROVIDER พิมพ์คำว่า gemini
GEMINI_API_KEY คีย์ปัญญาประดิษฐ์ขึ้นต้นด้วย AQ. หรือ AL. จากข้อ 1.1
GEMINI_MODEL พิมพ์คำว่า gemini-3.5-flash
BOT_NAME ชื่อบอตที่จะใช้เรียกในระบบ เช่น TaskyBot
SESSION_SECRET พิมพ์ตัวอักษรสุ่มยาวๆ เพื่อความปลอดภัยล็อกอิน Dashboard
CRON_SECRET พิมพ์ตัวอักษรสุ่มยาวๆ เพื่อความปลอดภัยในการเรียกฟังก์ชันระบบเวลา
  1. เมื่อใส่ครบถ้วนเรียบร้อยแล้ว ให้คลิกปุ่มสีน้ำเงิน "Deploy"
  2. รอระบบ Compile ประมาณ 1-2 นาที เมื่อหน้าจอแสดงผลเสร็จเรียบร้อย ให้คัดลอกลิงก์ที่อยู่ URL ที่ Vercel ให้มา (เช่น https://jira-googlechat-bot.vercel.app)

3. การกำหนดค่าและตั้งค่า Webhooks (จุดสำคัญที่สุด)

ขั้นตอนนี้เป็นการเชื่อมต่อสายส่งข้อมูลเพื่อให้ระบบภายนอกสื่อสารกับแอปพลิเคชันหลังบ้านของเราได้:

3.1 ตั้งค่าเว็บฮุคฝั่ง Jira Cloud (ซิงก์ข้อมูลเวลามีตั๋วงานเคลื่อนไหว)

  1. เปิดเบราว์เซอร์แล้วเข้าใช้งานลิงก์ทางลัดนี้: https://[JIRA_DOMAIN_ของคุณ]/plugins/servlet/webhooks
  2. คลิกปุ่ม **"Create a Webhook"** ที่มุมขวาบน
  3. กำหนดรายละเอียดดังนี้:
    • Name: ตั้งชื่อตามต้องการ เช่น Sync to Dashboard
    • URL: นำลิงก์เว็บ Vercel ของท่านมากรอกและเติมท้ายลิงก์ด้วยคำว่า /api/jira/webhook เสมอ
      ตัวอย่าง: https://jira-googlechat-bot.vercel.app/api/jira/webhook
    • Events (หัวข้อส่งสัญญาณ): เลื่อนลงไปที่แถบ **Issue** และติ๊กเครื่องหมายเลือก 3 ตัวนี้:
      ☑️ **Created** (ส่งเมื่อสร้างตั๋ว)
      ☑️ **Updated** (ส่งเมื่อแก้ไขงานหรือเปลี่ยนดิวเดต/ย้ายสถานะบอร์ด)
      ☑️ **Deleted** (ส่งเมื่อมีการกดลบตั๋วงาน)
  4. กดบันทึกโดยคลิกปุ่ม **"Create"** ด้านล่างสุด

3.2 ตั้งค่าแอปพลิเคชันห้องแชท (Google Chat App)

สำหรับกำหนดทิศทางให้เวลาเราพิมพ์ `@ชื่อบอต` สั่งงานในห้องแชท แล้วห้องแชทยิงคำถามส่งมาหาสมอง AI ที่ Vercel:

  1. เข้าหน้าเว็บ **[Google Cloud Console](https://console.cloud.google.com/)** เลือกโปรเจกต์ระบบให้ถูกต้อง
  2. ค้นหาเปิดบริการ **"Google Chat API"** และกดปุ่ม **"Enable"** (เปิดใช้งาน)
  3. คลิกเมนู **"Configuration" (การกำหนดค่า)** แถบเมนูด้านซ้ายมือ
  4. กรอกข้อมูลบอตเบื้องต้น เช่น App name, Avatar URL, Description ให้สวยงาม
  5. เลื่อนลงมาที่หัวข้อ **Connection settings** ติ๊กเลือกที่ **"Webhook URL"**
  6. นำลิงก์แอป Vercel ของท่านมากรอกลงในช่อง และต้องพิมพ์ต่อท้ายลิงก์ด้วยคำว่า /api/webhook เสมอ
    ตัวอย่าง: https://jira-googlechat-bot.vercel.app/api/webhook
  7. ในหัวข้อ **Visibility** เลือกติ๊กจำกัดสิทธิ์ใช้งานเฉพาะบัญชีในองค์กรของท่าน
  8. กดปุ่ม **"Save" (บันทึก)** บันทึกข้อมูล

4. การเริ่มระบบและซิงก์ข้อมูลดึงตั๋วงานครั้งแรก (First Sync)

เพื่อให้ตัวบอตและหน้า Dashboard รู้จักประวัติรายชื่อพนักงานและดึงงานเก่าที่เคยสร้างค้างไว้ใน Jira มาเก็บในระบบ:

  1. เปิดหน้าเว็บแดชบอร์ดงานขึ้นมา (เช่น https://jira-googlechat-bot.vercel.app/dashboard)
  2. กดสมัครสมาชิกบัญชีแรกของระบบเพื่อใช้ล็อกอินเป็นบัญชีดูแลระบบ (Admin)
  3. จากนั้น ให้ทำการสั่งยิงข้อมูลอัปเดตงานทั้งหมดด้วยการเปิดลิงก์ดึงข้อมูลนี้บนแท็บหน้าเว็บเบราว์เซอร์:
    รูปแบบลิงก์: https://[ชื่อเว็บคลาวด์ของคุณ]/api/cron/sync-tickets?secret=[ค่า_CRON_SECRET_ของคุณ]
    ตัวอย่างลิงก์จริง: https://jira-googlechat-bot.vercel.app/api/cron/sync-tickets?secret=my_cron_secret_123
  4. หากขึ้นข้อความระบุว่า sync success แสดงว่าดึงข้อมูลเสร็จสิ้นเรียบร้อยแล้ว หน้าแดชบอร์ดบอร์ดและบอตก็พร้อมเริ่มตอบคำถามและแสดงสถิติงามๆ ทั้งหมดได้ทันทีครับ!

5. สรุปฟังก์ชันระบบและตั้งเวลาแจ้งเตือนค้าง (Vercel Cron)

ในตัวโปรเจกต์นี้ได้รับการตั้งค่าไฟล์กำหนดระบบเวลารันงานอัตโนมัติ (Vercel Cron Jobs) ผ่านไฟล์ vercel.json ไว้เรียบร้อยแล้วครับ ซึ่งจะรันหลังใช้งานจริงดังนี้:

  • การรายงานสรุปประจำวัน (Daily Summary): บอตจะแจ้งผลสรุปประสิทธิภาพงานประจำวันของทีมเข้าไปที่ห้องแชทอัตโนมัติ ทุกวันเวลา 18:00 น.
  • ระบบซิงก์ประวัติงาน (Hourly Ticket Sync): จะซิงก์ดึงความเคลื่อนไหวจาก Jira มารอไว้ในหน้าแดชบอร์ด ทุกๆ ชั่วโมงเพื่อความอัปเดตสดใหม่
